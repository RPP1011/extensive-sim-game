Tokenization

Tokenization converts raw text (or DSL code) into a sequence of integer IDs that a neural network can process. Each unique token (word, number, symbol) maps to an ID, and that ID indexes into a learnable embedding table — a lookup table of vectors.

Key terms:

  • Token — The smallest unit the model sees. Could be a word, subword, number, or symbol.
  • Vocabulary — The set of all known tokens. Our ability DSL has a 252-token vocabulary.
  • Token ID — Integer index into the vocabulary. “damage” → 42, “circle” → 87, etc.
  • Embedding table — A matrix of shape [vocab_size × d_model]. Row i is the learnable vector for token i.
  • [CLS] token — Special token prepended to the sequence. Its output embedding becomes a summary of the entire input.
  • [MASK] token — Used in masked language modeling. Replaces a token during training; the model must predict the original.
  • Padding — Sequences have different lengths. Shorter ones are padded with a special [PAD] token so batches are rectangular.

Residual Connections

A residual connection (skip connection) adds the input of a layer directly to its output: y = f(x) + x. This simple trick makes deep networks trainable. Without it, gradients vanish through many layers and the network can't learn. With it, the network only needs to learn the difference (residual) from the input.

Key terms:

  • Skip connection — Another name for residual connection. The input “skips” over the layer and is added to the output.
  • Residual — The difference f(x) = y − x. The layer learns “what to change” rather than “the complete answer.”
  • Identity shortcut — The skip path passes x unchanged. If the layer learns f(x)=0, the output is just x — the layer is a no-op. This is easier than learning the identity function from scratch.
  • Gradient highway — Gradients flow directly through the skip connection, bypassing the layer. This prevents vanishing gradients in deep networks.
  • Pre-norm residual — LayerNorm(x) → layer → + x. Used in modern transformers (including ours).

Autoencoder

An autoencoder is a neural network trained to reproduce its own input. It consists of an encoder that compresses the input to a small bottleneck (the latent space), and a decoder that reconstructs from it. The bottleneck forces the network to learn the most important features, discarding noise.

Key terms:

  • Bottleneck — The narrow middle layer. If the input is 80-dim and the bottleneck is 32-dim, the network must compress 80 numbers into 32.
  • Reconstruction — The decoder's output. A good autoencoder's reconstruction closely matches the original input.
  • Representation learning — The bottleneck representation is often more useful than the raw input. It captures structure, not noise.
  • Variational Autoencoder (VAE) — The bottleneck is a probability distribution, not a fixed vector. Enables generation of new data by sampling.
  • Denoising Autoencoder — Add noise to the input, train the network to reconstruct the clean version. Forces robust features.

Latent Spaces

A latent space is a compressed, learned representation of data. An encoder maps high-dimensional inputs (like an ability's 80 raw properties) into a low-dimensional embedding (like a 32-dim vector). Similar inputs cluster together in this space — not because we told them to, but because the network learns that similarity is useful for downstream tasks.

Key terms:

  • Embedding — A point in latent space. Each ability, entity, or token gets mapped to a dense vector.
  • Encoder — The network that compresses inputs into the latent space (high-dim → low-dim).
  • Decoder — Reconstructs the original from the embedding (low-dim → high-dim). Used during training to ensure the embedding preserves information.
  • Manifold — The geometric surface in latent space where valid data lives. Real data doesn't fill the whole space — it lies on a lower-dimensional manifold.
  • Reconstruction loss — MSE between the decoder's output and the original input. Forces the bottleneck to retain useful information.
  • Contrastive loss — Pulls same-category embeddings together and pushes different categories apart.
Bottleneck dimension: 32
