Actor-Critic

Actor-Critic splits the RL agent into two networks: the actor decides what to do (policy), and the critic evaluates how good the situation is (value function). The critic's feedback replaces the noisy episode returns of REINFORCE, giving the actor a stable learning signal at every timestep.

Key terms:

  • Actor (π) — The policy network. Takes a state, outputs action probabilities. This is what runs in the game.
  • Critic (V or Q) — The value network. Takes a state (or state+action), outputs a scalar “how good is this?” score.
  • Advantage (A) — A = Q(s,a) − V(s). How much better this action is compared to the average. Positive = better than expected.
  • TD error (δ) — δ = r + γV(s') − V(s). The critic's “surprise” signal. Used to update both actor and critic.
  • Shared backbone — Actor and critic often share early layers (entity encoder) and split at the end. Saves parameters and computation.
  • Baseline — The critic serves as a baseline for the actor's gradient. Reduces variance compared to pure REINFORCE.

Value & Q-Functions

A value function V(s) answers “how good is this state?” — the expected total future reward from state s. A Q-function Q(s,a) answers “how good is taking action a in state s?” Together, they let the agent evaluate situations and compare actions without running full simulations.

V(s) = E[∑t γt rt | s0=s]     Q(s,a) = E[∑t γt rt | s0=s, a0=a]

Key terms:

  • V(s) — State value — Expected return from state s, following the current policy. High V = good situation regardless of what you do next.
  • Q(s,a) — Action value — Expected return from taking action a in state s, then following policy. Lets you compare actions directly.
  • Advantage A(s,a) = Q(s,a) − V(s) — How much better action a is than average. The actor uses this to learn which actions are above-average.
  • Bellman equation — V(s) = r + γV(s'). The value of a state equals the immediate reward plus the discounted value of the next state. The foundation of all value-based RL.
  • TD learning — Update V(s) toward r + γV(s') after each step. Don't wait for the episode to end — bootstrap from the current estimate.
  • Target network — A frozen copy of the Q-network used in the Bellman target. Updated slowly to prevent oscillation.

On-Policy vs Off-Policy

On-policy algorithms learn from data collected by the current policy. When the policy changes, old data is thrown away. Off-policy algorithms can learn from data collected by any policy, including old versions. Off-policy is more sample efficient but requires corrections for the distribution mismatch.

Key terms:

  • On-policy — Data must come from the current policy π. After each update, old data is stale. Examples: REINFORCE, PPO, A3C.
  • Off-policy — Data can come from any policy μ. Old experiences remain useful. Examples: SAC, DQN, IMPALA.
  • Sample efficiency — How many environment interactions are needed to learn. Off-policy methods reuse data, so they need fewer interactions.
  • Distribution mismatch — The data was collected under μ but we're learning π. If π and μ differ too much, learning is unstable.
  • Importance sampling — Corrects for distribution mismatch by weighting samples by π(a|s)/μ(a|s). V-Trace uses truncated importance weights.
  • Replay buffer — Stores past transitions for reuse. The key data structure enabling off-policy learning.

Reward Shaping

In RL, the agent learns from rewards. A sparse reward (win/lose at the end) gives almost no signal for thousands of intermediate actions. Reward shaping adds dense intermediate rewards that guide the agent toward good behavior, without changing the optimal policy.

Key terms:

  • Sparse reward — Signal only at the end: +1 for win, −1 for loss. Hard to learn from: which of 500 actions caused the win?
  • Dense reward — Signal every timestep: +0.1 for dealing damage, −0.05 for taking damage. Easier to learn but harder to design correctly.
  • Potential-based shaping — F(s,s') = γΦ(s') − Φ(s). Provably doesn't change the optimal policy — just speeds up learning.
  • Reward hacking — The agent finds a way to get high reward without doing what you intended. A common failure of poorly designed dense rewards.
  • Credit assignment — The fundamental RL problem: which past action caused this reward? Sparse rewards make this extremely hard.
  • Discount factor (γ) — How much to value future rewards vs immediate ones. γ=0.99 means a reward 100 steps from now is worth 0.99¹&sup0;&sup0; ≈ 0.37 of its face value.

Replay Buffer

A replay buffer (experience replay) stores past transitions (state, action, reward, next_state) so the agent can learn from them multiple times. Instead of learning from whatever just happened, the agent samples random mini-batches from the buffer. This breaks temporal correlations and dramatically improves stability.

Key terms:

  • Transition (s, a, r, s') — One step of experience: state, action taken, reward received, next state.
  • Buffer capacity — Maximum number of transitions stored. Typical: 100K–1M. When full, oldest transitions are overwritten.
  • Uniform sampling — Each stored transition has equal probability of being sampled. Simple and effective.
  • Prioritized replay — Sample transitions with high TD error more often — focus on “surprising” experiences the model hasn't learned yet.
  • Decorrelation — Consecutive transitions are highly correlated (same episode). Random sampling breaks this correlation, which prevents the network from overfitting to recent trajectories.
  • Staleness — Old transitions were collected under a different policy. The further back, the less relevant. This is why off-policy corrections (importance sampling) matter.
