Probability Distribution

A probability distribution assigns a probability to every possible outcome. For discrete outcomes (like choosing an action), this is a PMF (Probability Mass Function) — a list of probabilities that must sum to 1. For continuous values, it's a PDF (Probability Density Function) that integrates to 1.

PMF: ∑i P(xi) = 1  |   PDF: ∫ p(x) dx = 1

Key terms:

  • PMF (Probability Mass Function) — Assigns a probability to each discrete outcome: P(attack)=0.4, P(heal)=0.3, etc.
  • PDF (Probability Density Function) — For continuous variables; probability of an interval is the area under the curve.
  • Uniform distribution — All outcomes equally likely. Maximum uncertainty — the agent has no preference.
  • Peaked distribution — One action dominates. The agent is confident about what to do.
  • Support — The set of outcomes with non-zero probability. A policy's support is all actions it might take.

Expectation (Expected Value)

The expected value E[X] is the weighted average of all possible outcomes, where each outcome is weighted by its probability. It appears everywhere in RL: expected return, expected gradient, expected advantage. It's the theoretical average you'd get over infinitely many samples.

E[X] = ∑i xi · P(xi)

Key terms:

  • E[X] — Expected value of random variable X. The “center of mass” of a distribution.
  • Weighted average — Unlike a plain average, each value is scaled by how likely it is.
  • Linearity — E[aX + bY] = aE[X] + bE[Y]. This makes expectations easy to work with algebraically.
  • Sample mean — The empirical estimate of E[X] from finite samples: (1/N)∑xi.

Log Probability

Log probabilities are fundamental to ML because they turn products into sums. When computing the probability of a sequence of independent events, you multiply probabilities — which quickly underflows to zero. Taking logs converts this to addition, keeping numbers in a stable range. The gradient of log π is the score function, the backbone of policy gradient methods.

log P(A,B) = log P(A) + log P(B)  |   ∇ log π = ∇π / π

Key terms:

  • log P — Natural logarithm of probability. Always ≤ 0 since 0 ≤ P ≤ 1. More negative = less likely.
  • Numerical stability — P=0.001×0.002×0.003 underflows; log versions sum to manageable numbers.
  • Score function — ∇ log π(θ) = ∇π(θ)/π(θ). The direction that increases the log probability of the chosen action.
  • Log-likelihood — The sum of log probabilities over a dataset. Maximized during supervised training.

Sampling

Sampling is drawing random values from a probability distribution. In RL, the policy outputs a distribution over actions, and we sample from it to decide what to do. Stochastic sampling enables exploration (trying suboptimal actions to discover better strategies), while deterministic selection (argmax) is pure exploitation.

Key terms:

  • Sample — A single draw from a distribution. Each draw is random but follows the distribution's shape.
  • Stochastic policy — Samples actions randomly according to π(a|s). Used during training for exploration.
  • Deterministic (greedy) — Always picks the most likely action: argmax π(a|s). Used at deployment.
  • Exploration vs exploitation — Sampling explores; argmax exploits. Temperature controls the tradeoff.
  • Histogram — As you collect more samples, their frequencies converge to the true distribution (law of large numbers).

Entropy

Entropy measures the uncertainty or “spread” of a probability distribution. A uniform distribution (all actions equally likely) has maximum entropy — maximum uncertainty. A deterministic distribution (one action with probability 1) has zero entropy. SAC maximizes entropy alongside reward to encourage exploration.

H(X) = −∑i pi log(pi)

Key terms:

  • H(X) — Entropy in nats (natural log) or bits (log base 2). Always ≥ 0.
  • Max entropy — Achieved by the uniform distribution: H = log(N) for N outcomes.
  • Min entropy — Zero, when one outcome has probability 1 (no uncertainty).
  • Entropy bonus — Added to the RL objective to prevent premature convergence: J = E[r] + αH(π).
  • Information content — −log(p) for a single outcome. Rare events carry more information.

KL Divergence

KL divergence measures how different one probability distribution is from another. It's the “extra surprise” you experience if you believe the distribution is Q but the true distribution is P. It's asymmetric: DKL(P||Q) ≠ DKL(Q||P). Used in PPO (implicit trust region) and V-Trace (importance weight clipping).

DKL(P || Q) = ∑i pi log(pi / qi)

Key terms:

  • DKL(P||Q) — KL divergence from Q to P. Always ≥ 0. Zero only when P = Q.
  • Asymmetry — DKL(P||Q) ≠ DKL(Q||P). It's not a true distance metric.
  • Forward KL — DKL(P||Q): penalizes Q for missing modes of P (mean-seeking).
  • Reverse KL — DKL(Q||P): penalizes Q for placing mass where P doesn't (mode-seeking).
  • Trust region — PPO implicitly constrains KL between old and new policy via clipping.

Variance

Variance measures how spread out values are around their mean. In RL, high variance in gradient estimates means noisy updates that make training unstable. REINFORCE has notoriously high variance because it uses full episode returns. Baselines, critics, and GAE all exist to reduce variance.

Var(X) = E[(X − μ)2]  |   σ = √Var(X)

Key terms:

  • Var(X) — Expected squared deviation from the mean. Units are squared, so we often use σ = √Var.
  • σ (standard deviation) — Square root of variance. In the same units as the data.
  • High variance gradients — REINFORCE multiplies logπ by episode return — returns vary wildly between episodes.
  • Baseline subtraction — Subtract E[G] to center the advantage around zero. Reduces variance without changing bias.
  • Critic (value function) — Learns V(s) to provide a low-variance baseline, giving us the advantage A = G − V(s).

Bias-Variance Tradeoff

In RL, we face a fundamental tradeoff: TD learning (one-step bootstrap) has low variance but is biased because it relies on the value function estimate. Monte Carlo returns are unbiased but have high variance because they depend on the entire trajectory. GAE(γ, λ) smoothly interpolates between these extremes.

GAE: Â = ∑l=0∞ (γλ)l δt+l  |   λ=0 → TD   λ=1 → MC

Key terms:

  • Bias — Systematic error from using an approximation (the value function). TD is biased because V(s) isn't perfect.
  • Variance — Random noise from sampling. MC has high variance because returns depend on many random actions.
  • λ (lambda) — GAE tradeoff parameter. λ=0 is pure TD (low variance, some bias). λ=1 is pure MC (no bias, high variance).
  • λ=0.95 — Common default. Mostly Monte Carlo but with a touch of bootstrapping for stability.
  • δt — TD error: rt + γV(st+1) − V(st). The one-step advantage estimate.

Dot Product

The dot product a·b = ∑ aibi measures how aligned two vectors are. Positive means they point in the same direction, zero means orthogonal (unrelated), negative means opposite. It's the core operation in attention (Q·K), similarity computation, and every neural network layer (w·x).

a · b = ∑i aibi = |a| |b| cos(θ)

Key terms:

  • Algebraic form — Sum of element-wise products: a1b1 + a2b2 + …
  • Geometric form — |a||b|cos(θ): product of magnitudes times cosine of the angle between them.
  • Positive dot product — Vectors point in similar directions. In attention: high compatibility.
  • Zero dot product — Orthogonal vectors. No relationship.
  • Projection — (a·b)/|b| gives the length of a's shadow onto b. This is how attention “reads” values.

Matrix Multiplication

Matrix multiplication Wx + b is the fundamental operation of neural networks. Each output neuron computes a dot product of one weight row with the entire input vector, then adds a bias. The matrix W defines a linear transformation — it rotates, scales, and shears the input space.

yi = ∑j Wij xj + bi  |   y = Wx + b

Key terms:

  • Linear layer — y = Wx + b. The basic building block. A (3×4) weight matrix transforms 4 inputs into 3 outputs.
  • Each row is a detector — Row i of W defines what input pattern produces a high output yi.
  • Bias (b) — Shifts the output. Without bias, the transformation always maps zero to zero.
  • Parameters — An m×n matrix has m×n weights + m biases. A 32×30 layer has 990 learnable parameters.
  • Linear transformation — Preserves addition and scaling. Non-linearity (ReLU, GELU) is needed between layers for the network to learn non-linear functions.

Cosine Similarity

Cosine similarity is the dot product normalized by vector magnitudes: it measures the angle between two vectors, ignoring their lengths. Values range from −1 (opposite) through 0 (orthogonal) to +1 (identical direction). Used to compare embeddings — e.g., “is this ability similar to Fireball?”

cos(θ) = a · b / (|a| · |b|)  |   range: [−1, +1]

Key terms:

  • cos(θ) = +1 — Vectors point in the same direction. Embeddings are maximally similar.
  • cos(θ) = 0 — Orthogonal. The embeddings are unrelated.
  • cos(θ) = −1 — Opposite directions. Maximally dissimilar.
  • Length invariance — Unlike dot product, cosine similarity is unaffected by vector magnitudes. Only direction matters.
  • L2 normalization — If vectors are unit-length (|a|=|b|=1), cosine similarity equals the dot product. Our ability encoder uses L2-normalized embeddings.

Exponential Moving Average (EMA)

An EMA smooths a noisy signal by blending each new value with the running average. It gives more weight to recent values (controlled by α) while retaining memory of the past. Used in target networks (soft update), Grokfast (gradient filtering), and optimizer momentum (Adam's β parameters).

EMAt = α · xt + (1 − α) · EMAt−1

Key terms:

  • α (smoothing factor) — Weight for the new value. α=1 means no smoothing (raw signal). α→0 means heavy smoothing (slow response).
  • Target network — In DQN/SAC, a slowly-updated copy of the value network: θtarget ← τθ + (1−τ)θtarget.
  • Grokfast — Filters gradients with EMA to amplify slow-varying components that drive generalization.
  • Adam β1, β2 — EMA of gradients (momentum) and squared gradients (adaptive learning rate).
  • Exponential decay — Past values contribute with exponentially decreasing weight: (1−α)k for k steps ago.
