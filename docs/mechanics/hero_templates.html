Hero Templates & Classes

Heroes are defined as hybrid TOML stat blocks + .ability DSL files. 27 hand-crafted base heroes and 172 auto-imported LoL champions provide a deep roster for combat simulation.

Template Format

Each hero is defined by two files living in assets/hero_templates/:

  1. TOML file (hero_name.toml) — Stats, metadata, and ability slot bindings
  2. Ability file (hero_name.ability) — Ability definitions in the DSL

The TOML file is the entry point. It references the ability file and defines the hero's base stat block.

Example: TOML Stat Block

[hero]
name = "Iron Sentinel"
role = "tank"

[stats]
max_hp = 1200
attack_damage = 45
attack_range = 1.5
attack_cooldown_ms = 1200
move_speed_per_sec = 3.0
armor = 60.0
magic_resist = 40.0
max_resource = 100
resource_regen_per_sec = 5.0

[abilities]
file = "iron_sentinel.ability"

Stat scaling: Ability effects scale off unit stats via ScalingTerms. A mage's damage scales with ability_damage, a fighter's with attack_damage. The stat block defines these base values.

Ability Slots

Each hero has a Vec<AbilitySlot> populated from the .ability file. An AbilitySlot is the runtime wrapper around an AbilityDef:

pub struct AbilitySlot {
    pub def: AbilityDef,
    pub cooldown_remaining_ms: u32,
    pub charges: u32,
    pub active: bool,
    // ...
}

Heroes also have Vec<PassiveSlot> for passive abilities. See Passives & Triggers.

Content Library

27 Base Heroes

Hand-crafted heroes in assets/hero_templates/, each with a unique kit designed to test specific game mechanics. These cover the core archetypes: tanks, healers, mages, assassins, fighters, marksmen, and controllers.

172 LoL Champion Imports

Auto-generated hero templates in assets/lol_heroes/, imported from League of Legends champion data. These provide a massive roster for stress-testing the effect system and AI decision pipeline. Each import maps LoL champion abilities to the native effect system.

Source of truth: The .ability files are the canonical source for ability definitions. The TOML stat block only defines base stats and references the ability file.
