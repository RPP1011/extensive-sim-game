Delivery Methods

Seven ways effects can reach their targets — from instant application to persistent zones on the ground. Each method creates unique timing, counterplay, and tactical depth.

Overview

The Delivery dimension determines how an ability's effects reach their targets. It's one of the five composable dimensions of the effect system. Every ability has an optional delivery; if none is specified, effects are applied instantly.

pub enum Delivery {
    Instant,
    Projectile { speed, pierce, width, on_hit, on_arrival },
    Channel { duration_ms, tick_interval_ms },
    Zone { duration_ms, tick_interval_ms },
    Tether { max_range, tick_interval_ms, on_complete },
    Trap { duration_ms, trigger_radius },
    Chain { max_bounces, bounce_range, damage_falloff },
}

Instant

Delivery::Instant

Effects are applied immediately when the cast completes. No travel time, no delay. This is the default when no delivery is specified on an AbilityDef. Most targeted heals, buffs, and point-blank abilities use instant delivery.

Projectile

Delivery::Projectile {
    speed: f32,                        // Units per second
    pierce: bool,                      // Continue through first target?
    width: f32,                        // Collision width (skillshot)
    on_hit: Vec<ConditionalEffect>,    // Effects on hitting a unit
    on_arrival: Vec<ConditionalEffect>, // Effects at destination
}

Creates an in-flight projectile that travels from the caster toward the target or position. Projectiles are advanced each tick during the advance_projectiles phase of the combat loop.

  • Speed: Distance traveled = speed * dt_ms / 1000 per tick
  • Pierce: If true, the projectile continues past the first target it hits, potentially hitting multiple units
  • Width: For skillshot-style projectiles; wider projectiles are easier to hit but harder to dodge
  • on_hit: Additional effects applied to each unit the projectile strikes
  • on_arrival: Effects applied at the final destination (for ground-targeted projectiles)

Events: ProjectileSpawned, ProjectileHit, ProjectileArrived

ProjectileBlock: The ProjectileBlock effect blocks enemy projectiles in an area for its duration, providing a defensive counter.

Channel

Delivery::Channel {
    duration_ms: u32,       // Total channel time
    tick_interval_ms: u32,  // Apply effects every N ms
}

The caster enters a channeling state and applies effects repeatedly at the tick interval. The channel occupies the caster — they cannot move or use other abilities while channeling.

  • Channels are interruptible: any hard CC (stun, silence, knockback) or death cancels the channel
  • Tracked via ChannelState on the caster's UnitState
  • Advanced during tick_channels phase of the combat loop
  • Total effect = (duration_ms / tick_interval_ms) applications

Zone

Delivery::Zone {
    duration_ms: u32,       // How long the zone persists
    tick_interval_ms: u32,  // Apply effects every N ms
}

Places a persistent area on the ground at the target location. The zone lasts for its duration and applies effects to units inside it at each tick interval. The zone's shape is determined by the ability's Area parameter.

  • Zones are independent of the caster — they persist even if the caster dies
  • Ticked during tick_zones phase of the combat loop
  • Fires ZoneCreated event when placed
  • Counterplay: walk out of the zone area

Tether

Delivery::Tether {
    max_range: f32,                     // Break distance
    tick_interval_ms: u32,              // Apply effects every N ms (default 0)
    on_complete: Vec<ConditionalEffect>, // Effects when tether expires naturally
}

Creates a connection between the caster and the target. As long as the tether holds, effects are applied at the tick interval. The tether has two possible outcomes:

  1. Target stays in range: Tether completes naturally, on_complete effects fire (often a stronger effect like a stun)
  2. Target moves beyond max_range: Tether breaks early, no completion bonus

Tethers are checked during tick_tethers in the combat loop. This creates a strategic choice: stay close and eat the full effect, or burn movement/dash to break it.

Trap

Delivery::Trap {
    duration_ms: u32,       // How long the trap persists before expiring
    trigger_radius: f32,    // Detection radius
}

Places an invisible trap on the ground. When an enemy unit enters the trigger_radius, the trap activates and applies its effects. Traps that are not triggered expire after duration_ms.

  • Traps are invisible to the enemy team
  • Only triggers on enemy units (respects team allegiance)
  • Single-use: triggers once, then is consumed

Chain

Delivery::Chain {
    max_bounces: u32,       // Maximum number of bounces
    bounce_range: f32,      // Max distance for next bounce target
    damage_falloff: f32,    // Multiplier per bounce (e.g., 0.8 = 20% less each bounce)
}

Effects bounce from the initial target to nearby enemy units. Each bounce searches for the closest un-hit enemy within bounce_range. Damage (and other numeric effects) are multiplied by damage_falloff with each bounce.

  • Fires ChainBounce event for each bounce with the bounce number
  • A chain with max_bounces: 5 and damage_falloff: 0.8 deals 100%, 80%, 64%, 51%, 41% on successive hits
  • Counterplay: spread out so bounces can't reach the next target
