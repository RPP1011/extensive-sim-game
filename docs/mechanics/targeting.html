Targeting & Area Shapes

How abilities select their targets. Every ability combines a targeting mode (who picks the target), an area shape (how many units are hit), and a range check (how far the caster can reach).

1. Targeting Modes

The AbilityTargeting enum on every AbilityDef determines what input the player or AI must provide when casting. There are 8 modes:

ModeInput RequiredDescription
TargetEnemy Enemy unit Click an enemy unit. The most common targeting mode for offensive abilities. The cast fails if the target is out of range or not visible.
TargetAlly Allied unit Click a friendly unit. Used for heals, shields, and buffs. The caster can target themselves unless the area is restricted.
SelfCast None Immediately targets the caster. No click required. Typical for self-buffs, defensive steroids, and transformations.
SelfAoe None Centered on the caster with an area shape. Hits all valid units in the area around the caster without requiring a target click. Used for auras and radial bursts.
GroundTarget Position (x, y) Click a point on the ground. The ability resolves at that position using the specified area shape. Common for skillshot AoEs and zone placement.
Direction Position (angle) Click a direction from the caster. Used for linear skillshots and cone abilities. The target position defines the direction vector from the caster.
Vector Start + direction Click-drag to define a start point and a direction. Used by abilities like Rumble R and Viktor E that place an effect along a player-defined line.
Global None Hits all valid units on the entire map regardless of range. No targeting input needed. Used by ultimates like Karthus R and Soraka R.

Default: If no targeting mode is specified in the ability definition, it defaults to TargetEnemy.

2. Area Shapes

The Area enum determines the shape of the affected region once a target or position has been selected. There are 7 shapes:

SingleTarget

Hits exactly one unit. When the ability target is a unit ID, that unit is hit. When the target is None, the caster is hit. No parameters.

area: SingleTarget

Circle

All valid units within a circular radius of the target point are hit. The center is determined by the ability target: a position, a unit's location, or the caster's location if no target is given.

ParameterTypeDescription
radiusf32Radius of the circle in world units.
 
area: Circle { radius: 3.5 }

Cone

A fan-shaped area originating from the caster, aimed toward the target. Units within the radius and within the angular spread are hit. The direction is computed as the vector from caster to target.

ParameterTypeDescription
radiusf32Maximum reach of the cone.
angle_degf32Total angular width of the cone in degrees (split evenly on each side of the direction vector).
 
area: Cone { radius: 5.0, angle_deg: 60.0 }

Line

A rectangular area extending from the caster toward the target. Units are hit if their perpendicular distance from the line's axis is within half the width, and their projected distance along the axis is between 0 and the line's length.

ParameterTypeDescription
lengthf32How far the line extends from the caster.
widthf32Total width of the rectangle (half on each side).
 
area: Line { length: 8.0, width: 1.5 }

Ring

An annular (donut-shaped) region. Hits units whose distance from the center is between the inner and outer radii. Useful for abilities that have a dead zone at the center.

ParameterTypeDescription
inner_radiusf32Inner boundary. Units closer than this are not hit.
outer_radiusf32Outer boundary. Units farther than this are not hit.
 
area: Ring { inner_radius: 2.0, outer_radius: 5.0 }

SelfOnly

Always resolves to exactly the caster, regardless of the ability target. No parameters. Distinct from SingleTarget with no target in that it always means the caster, even if a target was provided.

area: SelfOnly

Spread

Like Circle, but with an optional cap on the number of targets hit. Units within the radius are collected, then truncated to max_targets if the cap is non-zero.

ParameterTypeDescription
radiusf32Radius of the area.
max_targetsu32Maximum number of units to hit. 0 means unlimited (behaves like Circle).
 
area: Spread { radius: 4.0, max_targets: 3 }

Default: If no area is specified on an effect, it defaults to SingleTarget.

3. Target Resolution

When an ability is cast, the engine calls resolve_targets() to convert the targeting mode, area shape, and raw AbilityTarget input into a concrete list of unit IDs that will receive the effect.

The function lives in sim/targeting.rs and follows these steps:

  1. Determine ally vs. enemy: The effect type is inspected. Heals, shields, buffs, resurrects, lifesteal, reflect, immunity, stealth, redirect, and link effects target allies. Everything else targets enemies.
  2. Compute the center/direction: Based on the AbilityTarget variant (Unit(id), Position(x,y), or None), the engine resolves the reference point. For directional shapes (Cone, Line), the vector from caster to target defines orientation.
  3. Filter by shape: Each Area variant runs its own geometric filter:
    • Circle / Spread — distance check from center
    • Cone — distance check + dot-product angle check against the half-angle
    • Line — projection along axis (0..length) + perpendicular distance (≤ half width)
    • Ring — distance between inner_radius and outer_radius
  4. Filter by team: Allies or enemies only, based on step 1.
  5. Filter by status: Stealthed enemies are skipped (they cannot be targeted by AoE). Banished units are skipped entirely. Dead units are skipped unless the effect is Resurrect.
  6. Cap targets: For Spread, if max_targets > 0, the list is truncated.

Stealth interaction: The units_in_radius() helper automatically filters out stealthed enemies. This means AoE abilities will not reveal or hit stealthed units.

4. Ability Range

Every AbilityDef has a range: f32 field (default 0.0) that defines the maximum distance at which the ability can be cast. Range is checked before the cast begins:

  • For TargetEnemy and TargetAlly, the distance from the caster to the target unit must be ≤ range.
  • For GroundTarget, the distance from the caster to the clicked position must be ≤ range.
  • For Direction and Vector, range defines the maximum extent of the resulting line or zone.
  • For SelfCast, SelfAoe, and Global, range is not checked (the ability always succeeds from any position).

The range field is separate from the area's radius. An ability can have a long range (the caster can be far away) but a small area of effect, or vice versa.

// Example: long-range small AoE
AbilityDef {
    name: "Meteor Strike",
    targeting: GroundTarget,
    range: 12.0,           // can cast from 12 units away
    effects: [..],
    delivery: Some(Projectile { speed: 15.0, .. }),
    // area on the effect:
    // Circle { radius: 2.5 }  -- only hits units within 2.5 of impact
}
