Abilities & Effects

52 effect types across 7 design phases, composed along five orthogonal dimensions. Every ability in the game is built from these primitives.

The Five Dimensions

Every ability is a composition of five independent dimensions:

DimensionQuestionTypeDetails
EffectWhat does it do?Effect enum (52 variants)See catalog below
AreaWhere does it hit?Area enum7 shapes
DeliveryHow does it reach?Delivery enum7 methods
TriggerWhen does it fire?Trigger enum20+ conditions
TagsWhat power level?TagsResistance, categories

Composability over inheritance. A single ability can combine multiple effects, any area shape, any delivery method, and conditional triggers. There is no class hierarchy of ability types — just flat data.

AbilityDef

The central type that defines an active ability:

pub struct AbilityDef {
    pub name: String,
    pub targeting: AbilityTargeting,  // How the ability is aimed
    pub range: f32,                   // Max cast range
    pub cooldown_ms: u32,             // Time between uses
    pub cast_time_ms: u32,            // Wind-up before effects apply
    pub ai_hint: String,              // Hint for AI decision-making
    pub effects: Vec<ConditionalEffect>, // What the ability does
    pub delivery: Option<Delivery>,   // How effects reach targets
    pub resource_cost: i32,           // Mana/resource cost
    // Advanced fields:
    // charges, toggle, recast, forms, evolve_into
}

Cooldowns

After use, an ability enters cooldown for cooldown_ms. Cooldowns tick down during the tick_hero_cooldowns phase of each tick. The cooldown_reduction buff stat accelerates this. The CooldownModify effect can directly reduce a specific ability's remaining cooldown.

Cast Times

A non-zero cast_time_ms means the unit enters a CastState and cannot act until the cast completes. Casts can be interrupted by hard CC. The CastStarted / AbilityCastStarted events fire at the start of a cast.

Resources

If resource_cost > 0, the unit must have sufficient resource (mana). See Units & Stats for the resource system (resource, max_resource, resource_regen_per_sec).

Conditional Effects

pub struct ConditionalEffect {
    pub effect: Effect,
    pub condition: Option<Condition>,  // Only apply if condition is met
    pub area: Option<Area>,            // Override ability-level area
    pub target: Option<AbilityTarget>, // Override target (self, allies, enemies)
}

Each ability can have multiple effects, each with optional conditions. This allows complex abilities like "deal damage to enemies, heal allies, and buff self" in a single cast.

The Ability DSL

Abilities are defined in .ability files using a custom DSL parsed by a winnow-based parser. The pipeline:

.ability file  →  Parser (winnow)  →  AST  →  lower.rs  →  AbilityDef

DSL Example

ability Fireball {
    target: enemy, range: 5.0
    cooldown: 5s, cast: 500ms
    cost: 15, hint: damage

    damage 55 [FIRE: 60]
}

ability FrostNova {
    target: self_aoe
    cooldown: 10s, cast: 300ms

    damage 20 in circle(3.0)
    stun 2s in circle(3.0) [CROWD_CONTROL: 80]
}

ability VampStrike {
    target: enemy, range: 1.5
    cooldown: 4s, cast: 200ms

    damage 35 + 50% caster_attack_damage [PHYSICAL: 50]
    heal 20 when hit_count_above(3)
}

Conditions

Each effect can have a when condition and an optional else clause:

damage 50 when target_hp_below(30%)
shield 40 when caster_hp_below(50%)
damage 30 when target_is_stunned else heal 20

Compound conditions: and(cond1, cond2), or(cond1, cond2), not(cond). Available conditions include target_hp_below, caster_hp_above, target_is_stunned, target_has_tag, hit_count_above, target_distance_below, ally_count_below, target_stack_count, and 20+ more.

Stacking & Probability

Stacking ModeBehavior
refresh (default)Reset duration on re-application
extendAdd duration
strongestKeep highest power level
stackAllow multiple instances up to max_stacks

Any effect can have a chance modifier: damage 50 chance 0.75 (75% trigger chance).

Advanced Features

  • Morph: morph into { ... } for 8s — temporarily replace the ability
  • Recast: recast: 1, recast_window: 10s — multi-cast within a window
  • Charges: charges: 6, recharge: 500ms — ammo system
  • Toggle: toggle, toggle_cost: 25.0 — on/off with resource drain
  • Form swap: form: "melee", swap_form: "ranged" — stance switching
  • Unstoppable: unstoppable — CC immune during cast

See the Ability DSL reference for full syntax documentation.
