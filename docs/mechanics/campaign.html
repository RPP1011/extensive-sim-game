Campaign Overworld

The top layer of the three-layer architecture. A turn-based overworld where factions compete for territory, the player's guild manages resources, and flashpoints escalate into missions.

The Three Layers

Campaign (you are here)
  ↓ flashpoints trigger
Mission (multi-room dungeons)     → see Missions
  ↓ rooms contain
Combat (100ms deterministic sim)  → see Combat Loop

The campaign layer is turn-based. Each turn, the player takes an action and all factions execute their AI. Events cascade downward — territorial conflicts generate flashpoints, flashpoints launch missions, missions resolve through combat.

The step_campaign Loop

pub fn step_campaign(
    state: &mut CampaignState,
    action: Option<CampaignAction>,
) -> CampaignStepResult

Each call to step_campaign advances one turn:

  1. Snapshot economy — Record gold, supplies, reputation before the turn for delta tracking
  2. Apply player action — Execute the player's chosen CampaignAction (if any)
  3. Tick factions — All NPC factions execute their AI via step_all_factions
  4. Generate events — Diplomatic shifts, territory changes, flashpoint creation
  5. Return result — CampaignStepResult with events and deltas

Campaign Actions

Each turn, the player can take one CampaignAction:

ActionDescription
DeploySend heroes to a region for defense or attack
RecruitSpend gold to add new heroes to the guild roster
TradeExchange resources between regions
DiplomacyImprove or worsen relations with a faction
RestRecover hero HP and cooldowns between missions
Accept MissionLaunch a mission from a flashpoint

Faction AI

Each faction has a FactionCommander that decides its action each turn via step_faction_turn:

pub fn step_faction_turn(
    faction: &FactionState,
    commander: &FactionCommander,
    map: &OverworldMap,
    diplomacy: &DiplomacyState,
    personality: Option<&FactionPersonalityConfig>,
    rng_seed: u64,
) -> FactionTurnResult

The AI uses score_actions to evaluate possible moves and picks the highest-scoring one. Personality configs bias the scoring — aggressive factions weight expansion, defensive factions weight fortification.

Deterministic faction AI: Like the combat layer, faction AI uses deterministic RNG (lcg_next) seeded from the map seed and turn number. Same map + same turns = identical faction behavior.

Regions & Territory

pub struct OverworldMap {
    pub regions: Vec<Region>,
    pub factions: Vec<FactionState>,
    pub map_seed: u64,
    // ...
}

The map is divided into regions, each owned by a faction. Regions provide resources and strategic value. Contested borders create flashpoints.

Diplomacy

Faction relationships are tracked in the DiplomacyState:

  • Allies: Won't attack each other, may cooperate
  • Neutral: No active conflict, can trade
  • Hostile: Actively compete for territory, generate flashpoints

Diplomatic relationships evolve based on faction actions (territory grabs shift relations hostile, trade shifts them positive).

Guild Economy

The player's guild manages three resources:

ResourceSourceUse
GoldControlled regions, mission rewardsRecruit heroes, buy equipment
SuppliesTrade, region productionSustain deployed heroes, mission upkeep
ReputationCompleted missions, diplomacyUnlock faction quests, improve recruitment

Economy deltas are tracked per-turn in StepDeltas so the player can see income vs. expenditure.

Flashpoints

When factions conflict over a region, a flashpoint is generated. Flashpoints are the bridge between campaign and missions:

  1. Hostile factions share a border
  2. AI decides to contest the region
  3. A flashpoint event fires
  4. The player can accept the flashpoint as a mission
  5. Mission resolves through combat
  6. Outcome feeds back into the campaign (territory change, reputation gain/loss)
