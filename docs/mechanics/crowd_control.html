Crowd Control

Twelve CC types ranging from hard lockdowns to soft movement restrictions. Each has distinct counterplay and strategic purpose in team fights.

Overview

Crowd control (CC) effects restrict a unit's ability to act, move, or use abilities. They are the primary tool for creating windows of opportunity in combat. CC effects are applied as status effects with a fixed duration that ticks down during the combat loop.

CC and the tick system: CC durations are tracked in milliseconds. Each 100ms tick decrements the remaining duration. A unit's CC status is checked at the start of per-unit processing — if CC is active, the unit skips its action phase.

Hard CC

Hard CC completely prevents the target from taking voluntary actions.

Stun

Effect::Stun { duration_ms: u32 }

The most common hard CC. Prevents all movement, abilities, and auto-attacks for the duration. The stunned unit is locked in place and cannot act. Fires ControlApplied event. The OnStunExpire trigger fires when the stun wears off.

Suppress

Effect::Suppress { duration_ms: u32 }

The strongest CC in the game. Like stun, but cannot be cleansed by normal means (Dispel, QSS-style effects). Only the suppressing unit's death or the duration expiring ends it. Typically used for ultimate abilities.

Banish

Effect::Banish { duration_ms: u32 }

Removes the target from play entirely. The unit becomes untargetable and unable to act. No damage can be dealt to or by the banished unit. Can be used defensively (save an ally) or offensively (remove a threat).

Polymorph

Effect::Polymorph { duration_ms: u32 }

Transforms the target, preventing all abilities and attacks but allowing movement. The polymorphed unit can still walk but cannot fight back.

Forced Action CC

These effects override the target's AI, forcing specific behavior.

Charm

Effect::Charm { duration_ms: u32 }

Forces the target to walk toward the caster. The charmed unit cannot attack or use abilities — it just walks harmlessly toward whoever charmed it.

Fear

Effect::Fear { duration_ms: u32 }

Forces the target to move in a random direction away from the caster. The feared unit cannot attack or use abilities.

Taunt

Effect::Taunt { duration_ms: u32 }

Forces the target to auto-attack the taunter. The taunted unit can move (toward the taunter) but cannot use abilities or attack anyone else. Key defensive tool for tanks.

Soft CC

Soft CC partially restricts the target without fully disabling them.

Root

Effect::Root { duration_ms: u32 }

Prevents all movement (walking, dashing) but the target can still attack and use non-movement abilities. The Grounded effect is automatically applied during root.

Silence

Effect::Silence { duration_ms: u32 }

Prevents all ability usage but the target can still move and auto-attack. Particularly devastating against ability-dependent heroes.

Grounded

Effect::Grounded { duration_ms: u32 }

Specifically prevents dashes, blinks, and movement abilities. Normal walking is still allowed. Counters high-mobility heroes.

Slow

Effect::Slow { factor: f32, duration_ms: u32 }

Tracked as a status effect with a speed reduction factor. A factor: 0.5 represents a 50% slow.

Implementation note: The Slow status is currently applied and tracked (visible to conditions and triggers) but does not mechanically reduce move_speed_per_sec. It functions as a marker for AI and conditional effects. Attack speed debuffs (debuff attack_speed) do work mechanically.

Blind

Effect::Blind { miss_chance: f32, duration_ms: u32 }

Causes auto-attacks to miss with the given probability. A miss_chance: 0.75 means 75% of attacks miss. Fires AttackMissed event on miss. Does not affect abilities.

Confuse

Effect::Confuse { duration_ms: u32 }

Randomizes the target's actions. Movement becomes random, and attacks/abilities may target allies instead of enemies.

Displacement

Displacement effects forcibly reposition units. They are instant and do not have a duration. See Movement & Positioning for details.

Countering CC

Immunity

Effect::Immunity { immune_to: Vec<String>, duration_ms: u32 }

Grants immunity to specific effect tags for the duration. When an immune unit would receive a matching effect, it's blocked and an EffectResisted event fires.

Dispel

Effect::Dispel { target_tags: Vec<String> }

Removes active status effects from a unit. Can be filtered by tags to only remove specific categories (e.g., only CC effects, only debuffs). Fires DispelApplied event with the count of removed effects.

Resistance Tags

Units have a resistance_tags: Tags field that can grant permanent immunity to specific effect categories. This is checked before any effect is applied.
