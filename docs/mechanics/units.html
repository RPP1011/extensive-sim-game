Units & UnitState

Complete reference for the UnitState struct — every field, team mechanics, death handling, and the casting system that drives all unit actions.

1. UnitState — Full Struct Reference

The UnitState struct is the core data representation for every entity on the battlefield. It is defined in crates/tactical_sim/src/sim/types.rs and is fully serializable via serde. Below is every field, grouped by function.

Identity

FieldTypeDescription
idu32Unique unit identifier. Used by intents, events, and targeting. Never changes once assigned.
teamTeamWhich side this unit fights for: Hero or Enemy. Determines targeting rules and win conditions.

Health

FieldTypeDescription
hpi32Current hit points. When this reaches 0 or below, the unit dies. Invariant: hp <= max_hp.
max_hpi32Maximum hit points. Heals cannot exceed this value.
shield_hpi32Temporary extra health from Shield effects. Absorbs damage before hp is reduced. Defaults to 0.

Combat — Basic Attacks

FieldTypeDescription
attack_damagei32Base damage dealt by auto-attacks.
attack_rangef32Maximum range (in world-units) for starting an attack. If the target is farther, the unit repositions first.
attack_cooldown_msu32Minimum time (ms) between auto-attacks. Determines attack speed.
attack_cast_time_msu32Wind-up time (ms) before an attack resolves. The unit is locked in a cast state during this time.
cooldown_remaining_msu32Time (ms) until the next auto-attack is available. Decremented each tick by dt_ms.

Abilities

FieldTypeDescription
ability_damagei32Base damage for the legacy single-ability system.
ability_rangef32Range of the legacy ability. Note: checked at 95% of stated range to prevent edge-case misses.
ability_cooldown_msu32Cooldown for the legacy ability.
ability_cast_time_msu32Cast time for the legacy ability.
ability_cooldown_remaining_msu32Remaining cooldown on the legacy ability.
abilitiesVec<AbilitySlot>The hero ability engine's ability list. Each slot contains an AbilityDef (parsed from .ability DSL files), current cooldown, and per-ability state. This is the primary ability system.
passivesVec<PassiveSlot>Passive abilities that trigger on events (e.g., OnDamageTaken, OnKill). Defined via the .ability DSL.

Healing

FieldTypeDescription
heal_amounti32Amount healed per cast (legacy heal system).
heal_rangef32Maximum range for targeting an ally to heal.
heal_cooldown_msu32Cooldown between heals.
heal_cast_time_msu32Wind-up time before the heal resolves.
heal_cooldown_remaining_msu32Remaining cooldown on the heal.

Crowd Control

FieldTypeDescription
control_rangef32Maximum range for applying the legacy CC ability.
control_duration_msu32How long the CC lasts on the target.
control_cooldown_msu32Cooldown between CC casts.
control_cast_time_msu32Wind-up time before CC resolves.
control_cooldown_remaining_msu32Remaining cooldown on the CC ability.
control_remaining_msu32Time (ms) this unit remains crowd-controlled (stunned). While > 0, the unit cannot act: casting is cancelled, and a UnitControlled event is emitted each tick. Unstoppable abilities ignore this.

Defense

FieldTypeDescription
armorf32Reduces physical damage. Formula: reduction = armor / (100 + armor). Default: 0.
magic_resistf32Reduces magic damage. Formula: reduction = mr / (100 + mr). Default: 0.
cover_bonusf32Damage reduction from nearby terrain cover. Range: 0.0 (none) to 0.5 (maximum). Updated per-tick from terrain context. See Movement & Positioning.
elevationf32Unit's height above baseline in metres. Updated per-tick from terrain. Affects combat bonuses and AI positioning decisions.
resistance_tagsTagsTag-based resistance map (e.g., {"fire": 2}). When an incoming effect has a matching tag with power <= resistance, the effect is resisted entirely and an EffectResisted event is emitted.

Movement

FieldTypeDescription
positionSimVec2Current position on the battlefield as { x: f32, y: f32 }. Invariant: both components must be finite. See Movement & Positioning.
move_speed_per_secf32Movement speed in world-units per second. Per-tick distance: speed * dt_ms / 1000.

Resource

FieldTypeDescription
resourcei32Current resource (e.g., mana, energy, rage). Consumed by abilities that have a resource cost.
max_resourcei32Maximum resource pool. Default: 0 (no resource system).
resource_regen_per_secf32Passive resource regeneration per second. Applied each tick.

State

FieldTypeDescription
castingOption<CastState>If Some, the unit is currently winding up an attack, ability, or heal. See CastState below.
channelingOption<ChannelState>If Some, the unit is channeling an ability. Channels tick periodically and can be interrupted by CC.
status_effectsVec<ActiveStatusEffect>All active buffs, debuffs, CC, DoTs, and HoTs on this unit. Each has a StatusKind, source ID, remaining duration, tags, and stacking info.
state_historyVecDeque<(u32, SimVec2, i32)>Ring buffer of (tick, position, hp) snapshots. Limited to the last 50 entries. Used by the Rewind effect to restore a unit to a previous state. Skipped in serialization.

Summon

FieldTypeDescription
owner_idOption<u32>If set, this unit was summoned by the owner. Used to track summon-master relationships.
directedboolIf true, this summon does not act independently. It attacks when its owner attacks, from its own position. Directed summons skip intent processing entirely during the main simulation loop.

Tracking

FieldTypeDescription
total_healing_donei32Cumulative healing output across the entire combat. Used by AI targeting priority to identify high-value healer targets. Skipped in serialization.
total_damage_donei32Cumulative damage output across the entire combat. Used by AI targeting priority to identify high-value damage dealer targets. Skipped in serialization.

2. Teams

The Team enum has two variants:

pub enum Team {
    Hero,
    Enemy,
}

Team membership determines:

  • Targeting: Offensive actions (Attack, Ability, Control) can only target units on the opposing team. Healing targets units on the same team.
  • Win condition: Combat ends when all units on one team are dead. The surviving team is recorded as the winner in SimMetrics.winner.
  • AI behavior: Squad AI, GOAP planners, and ability evaluators all use team membership to distinguish allies from enemies for tactical decisions.
  • Vision: The Skulk and Hide movement intents compute enemy vision cones by filtering units where team != unit_team.

Confuse is the one exception to strict team-based targeting. A confused unit selects a random alive unit (from either team, excluding itself) as its target, allowing friendly fire.

3. Death

A unit dies when hp <= 0. The simulation handles death in several stages:

Death Event

When damage reduces a unit's HP to zero or below, a UnitDied event is emitted:

SimEvent::UnitDied {
    tick: u64,
    unit_id: u32,
}

State Cleanup

At the end of every tick, the simulation sweeps all dead units and clears stale state:

  • casting is set to None
  • channeling is set to None
  • control_remaining_ms is set to 0

Status Effect Cleanup

When a unit dies, status effects on other living units that reference the dead unit are removed. This prevents stale references for effects like:

  • Taunt { taunter_id } — no longer forced to attack a dead taunter
  • Duel { partner_id } — duel ends when partner dies
  • Link { partner_id } — damage-sharing link broken
  • Redirect { protector_id } — damage redirection ends
  • Attached { host_id } — detach when host dies

Dead Units in Processing

Dead units are skipped during the per-unit processing loop (is_alive() check). They remain in the UnitStore for the duration of combat — they are not removed, allowing post-combat analysis and potential resurrection via the Resurrect effect.

4. CastState

When a unit begins an attack, ability, heal, or control action, a CastState is placed in the unit's casting field. The unit is locked into this cast until it completes or is interrupted.

pub struct CastState {
    pub target_id: u32,
    pub target_pos: Option<SimVec2>,
    pub remaining_ms: u32,
    pub kind: CastKind,
    pub area: Option<Area>,
    pub ability_index: Option<usize>,
    pub effect_hint: CastEffectHint,
}
FieldDescription
target_idThe unit being targeted by this cast. Used to resolve the effect when the cast completes.
target_posOptional ground-target position. Used by abilities with AbilityTarget::Position.
remaining_msTime left until the cast completes. Decremented by dt_ms each tick. When it reaches 0, resolve_cast() fires.
kindThe type of cast. See CastKind below.
areaArea of effect shape, populated from AbilityDef at cast start for hero abilities. Used by AI spatial awareness to predict effect zones.
ability_indexIndex into the caster's abilities Vec for HeroAbility casts. Allows the system to look up the full AbilityDef during resolution.
effect_hintSimplified effect category for AI decision-making. See CastEffectHint below.

CastKind

Identifies what type of action is being cast:

VariantDescription
AttackBasic auto-attack. Uses attack_damage and attack_cast_time_ms.
AbilityLegacy single-ability cast. Uses ability_damage.
HealLegacy heal cast. Uses heal_amount.
ControlLegacy CC cast. Applies control_duration_ms to the target.
HeroAbility(usize)Hero ability engine cast. The usize is the index into the unit's abilities Vec, linking to the full AbilityDef with all its effects, conditions, and delivery mechanisms.

CastEffectHint

A simplified classification of what an ability will do, derived from its effects at cast start. Used by the AI to make spatial decisions (e.g., should allies dodge? should enemies spread?).

HintMeaning
UnknownDefault. No classification available.
DamagePrimarily deals damage.
HealPrimarily heals.
CrowdControlApplies CC (stun, slow, root, knockback, pull, silence, polymorph, fear, grounded, suppress).
ObstacleCreates terrain obstacles.
BuffApplies shields, buffs, or stealth.
SummonSummons new units.
MixedCombines two or more of the above categories in a single ability.

Cast Interruption

Casts are interrupted (set to None) when the caster:

  • Has control_remaining_ms > 0 (stunned) — unless the ability is flagged as unstoppable
  • Is affected by Banish (removed from play)
  • Is affected by Polymorph (transformed, cannot use abilities or attacks)
  • Is affected by Fear (forced to flee)
  • Dies (hp <= 0)

Unstoppable abilities are special: when a unit is casting an ability with def.unstoppable = true and gets stunned, the CC is cleared instead of the cast. The ability continues uninterrupted.
