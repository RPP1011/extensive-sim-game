Damage & Defenses

Three damage types, two resistance stats, shields, cover, elevation, and a composable scaling system. Everything that determines how much HP a unit actually loses.

Damage Types

pub enum DamageType {
    Physical,  // Reduced by armor
    Magic,     // Reduced by magic_resist
    True,      // Ignores all resistances
}

The default damage type is Physical. Abilities specify their damage type in the effect definition.

TypeReduced ByUse Case
PhysicalArmorAuto-attacks, melee abilities, most AD-scaling skills
MagicMagic ResistSpells, magic abilities, AP-scaling effects
TrueNothingExecute effects, pure damage, percent-HP damage

Armor & Magic Resist

Both stats use the same reduction formula:

Damage Reduction Formula:

reduction = resistance / (100 + resistance)

actual_damage = raw_damage * (1 - reduction)

Where resistance is either armor (for Physical) or magic_resist (for Magic). This formula has diminishing returns — each point of resistance is slightly less effective than the last, but always provides positive benefit:

ResistanceReductionEffective HP Multiplier
00%1.00x
2520%1.25x
5033%1.50x
10050%2.00x
15060%2.50x
20067%3.00x
30075%4.00x

Each point of resistance effectively gives 1% more effective HP against that damage type. 100 armor means you have 200% effective HP against physical damage — you take 2x as many hits to kill.

// On UnitState:
pub armor: f32,         // Reduces physical damage
pub magic_resist: f32,  // Reduces magic damage

Damage Variance

All damage and healing have built-in variance to prevent perfectly predictable outcomes:

// Damage: 90-110% of base amount (uniform random)
let variance_percent = 90 + (next_rand_u32(state) % 21) as i32;
let damage = ((base_amount * variance_percent) + 99) / 100;

// Healing: 95-105% of base amount (tighter variance)
let variance_percent = 95 + (next_rand_u32(state) % 11) as i32;

This uses the deterministic RNG, so the same seed always produces the same variance rolls.

The Full Damage Pipeline

When damage is applied, it passes through these stages in order:

  1. Base variance — 90–110% random roll
  2. Resistance reduction — Armor (physical) or MR (magic). True bypasses.
  3. Damage output buff/debuff — Source's damage_output stat modifier
  4. Cover reduction — 1 - cover_bonus, scaled by outnumber ratio
  5. Elevation — ±15% per metre of height difference
  6. DamageModify status — Multiplicative factor on the target
  7. Redirect check — Transfer to protector if Redirect charges remain
  8. Shield absorption — Reduce shield_hp first, overflow to HP
  9. HP application — Reduce HP (clamped at 0)
  10. DeathMark accumulation — Track damage for eventual burst
  11. Reflect — Return damage * percent / 100 to source
  12. Lifesteal — Heal source for damage * percent / 100
  13. Stealth break — Remove stealth if break_on_damage
  14. Link sharing — Partner takes damage * share_percent / 100
  15. Death processing — If HP ≤ 0, fire OnDeath triggers

Shields

Shields provide temporary bonus HP that absorbs damage before the unit's actual HP is reduced.

Effect::Shield { amount: i32, duration_ms: u32 }

When a unit with shield_hp > 0 takes damage:

  1. Damage first reduces shield_hp
  2. Any overflow reduces hp
  3. ShieldAbsorbed event is emitted with the absorbed amount and remaining shield

Shield ordering: Shields are applied after armor/MR reduction. A shield absorbs the post-mitigation damage amount.

Advanced Shield Mechanics

EffectDescription
OverhealShieldHealing that exceeds max HP is converted to shield (default 100% conversion)
AbsorbToHealCreates a shield; when the shield expires or is broken, converts absorbed damage into healing (default 50%)
ShieldStealRemoves shield from the target and adds it to the caster

The OnShieldBroken trigger fires when a unit's shield reaches zero from damage.

Cover & Elevation

Cover Bonus

pub cover_bonus: f32,  // 0.0 (none) to 0.7 (max)

Units near terrain features receive a cover_bonus that reduces incoming damage. The effective cover is scaled by outnumber ratio — a unit fighting outnumbered gets more value from cover:

// Outnumber ratio: sqrt(enemies / allies), clamped 1.0-4.0
// 4v1 → 2.0x cover; 2v1 → 1.41x; 1v1 → 1.0x
effective_cover = (cover_bonus * sqrt(enemies / allies)).min(0.7);
final_damage = post_armor_damage * (1.0 - effective_cover);

True damage bypasses cover entirely.

Elevation

pub elevation: f32,  // Metres above baseline

Elevation provides a ±15% damage modifier per metre of height difference:

  • Attacker on high ground: +15% damage per metre above target
  • Defender on high ground: -15% damage per metre above attacker (amplified by outnumber ratio)
  • Total elevation modifier is clamped to 0.5x–1.3x

True damage bypasses elevation modifiers. Elevation is updated per-tick from the GridNav terrain context.

Damage Scaling

Ability damage scales with unit stats via ScalingTerms:

pub struct ScalingTerm {
    pub stat: String,     // e.g. "attack_damage", "ability_damage", "max_hp"
    pub percent: f32,     // Scaling percentage
}

// Total damage = base_amount + sum(unit.stat_i * percent_i / 100)

The Damage effect supports both a legacy single-stat scaling (scaling_stat + scaling_percent) and the newer composable bonus: Vec<ScalingTerm> system. New abilities should prefer bonus.

Example: An ability with amount: 80 and bonus: [{stat: "attack_damage", percent: 60}] on a unit with 100 AD deals 80 + (100 * 60/100) = 140 raw damage before resistances.

Damage over Time (DoT)

The Damage effect can deal periodic damage over time:

Effect::Damage {
    amount_per_tick: 20,     // Damage each tick
    duration_ms: 3000,       // Total duration (3 seconds)
    tick_interval_ms: 500,   // Ticks every 0.5s (6 ticks total)
    damage_type: Magic,
    ..
}

DoT effects are tracked as status effects and tick during the tick_status_effects phase of the combat loop.

Healing

Effect::Heal {
    amount: i32,             // Instant heal amount
    amount_per_tick: i32,    // HoT per tick
    duration_ms: u32,        // HoT duration
    tick_interval_ms: u32,   // HoT interval
    bonus: Vec<ScalingTerm>, // Stat scaling (same as damage)
    ..
}

Healing cannot exceed max_hp unless an OverhealShield effect is active. Healing fires HealApplied events and contributes to the unit's total_healing_done tracking stat (used for AI targeting priority).

Resurrect

Effect::Resurrect { hp_percent: f32 }

Brings a dead unit back to life at the specified HP percentage. Fires UnitResurrected event.
