Movement & Positioning

How units traverse the battlefield — walk speed, pathfinding, dashes, displacement effects, cover, and elevation within the 100ms fixed-tick simulation.

1. Move Speed

Every unit has a move_speed_per_sec field (in world-units per second). Each simulation tick, the maximum distance a unit can travel is computed as:

max_delta = move_speed_per_sec * (dt_ms / 1000.0)

With the standard fixed tick of FIXED_TICK_MS = 100, a unit with move_speed_per_sec = 4.0 moves up to 0.4 world-units per tick. This value is used by the move_towards_position() helper, which advances the unit along a straight line toward its target, clamped to max_delta.

Slow effects modify move speed via a Slow { factor, duration_ms } status effect. The factor is a multiplier applied to move_speed_per_sec during movement calculations (e.g., factor 0.5 = half speed). See Crowd Control for details.

After any movement, if a GridNav is present, the unit's elevation field is refreshed from the terrain data at the new position.

2. Pathfinding

Navigation uses a grid-based system via the GridNav struct. The grid divides the battlefield into square cells of configurable size, where each cell is either walkable or blocked. Pathfinding is handled in two stages:

Fast Path: Direct Movement

The simulation first tries a straight-line move toward the target. If the resulting position lands in a walkable cell, no pathfinding is needed.

A* Fallback

If the direct path lands in a blocked cell, the system computes a waypoint using A* search with an 8-directional neighbor expansion (cardinal + diagonal). Key properties:

  • Heuristic: Euclidean distance to the goal cell.
  • Slope cost: Each cell can have a slope_cost_multiplier (default 1.0) that increases traversal cost for elevated terrain.
  • Corner-cutting prevention: Diagonal moves are blocked if either adjacent cardinal cell is unwalkable, preventing units from slipping through wall corners.
  • Iteration cap: A* is limited to 500 iterations to prevent pathfinding stalls in complex mazes.
  • Clamping: If the computed step still lands in a blocked cell, clamp_step_to_walkable() backs off along the movement vector in 10% increments until a valid position is found.

When no GridNav is present (e.g., open-field scenarios), movement uses simple vector math with no collision detection.

Special Movement Intents

IntentBehavior
MoveTo { position } Standard movement toward a target position using pathfinding.
Skulk { objective } Concealment-aware movement. Computes enemy vision cones, then picks the next cell that balances progress toward the objective with avoiding enemy sight lines. Prefers cells near walls (high concealment score).
Hide Searches a 5-cell radius for the cell that breaks line of sight with the most enemies. Scores cells by: hidden-from count, concealment value, and proximity.

3. Dashes

The Dash effect provides instant repositioning for the caster. It is defined with four parameters:

ParameterTypeDescription
to_targetboolIf true, dash toward the target unit, stopping 0.5 units away.
distancef32Maximum dash distance in world-units (default: 2.0).
to_positionboolIf true, dash toward the ability's target position rather than a unit.
is_blinkboolIf true, this is an instant teleport that ignores terrain and the Grounded status.

Dash Resolution

When a Dash effect fires:

  1. Grounded check: Non-blink dashes are blocked by the Grounded status effect. If blocked, a DashBlockedByGrounded event is emitted and the effect is consumed without movement.
  2. Direction:
    • to_position: move toward the specified ground position.
    • to_target: move to within 0.5 units of the target unit.
    • Neither: dash away from the target (escape dash).
  3. Execution: The caster's position is updated instantly. A DashPerformed event is emitted with from and to coordinates (encoded as x100 integers).
SimEvent::DashPerformed {
    tick, unit_id,
    from_x100, from_y100,
    to_x100, to_y100,
}

4. Knockback & Pull

These are displacement effects that force a target unit to a new position. Both operate on the same distance-based mechanic but in opposite directions.

Knockback

Knockback { distance: f32 } pushes the target away from the caster. The target's position is updated by calling move_away(target_pos, caster_pos, distance), which moves the target along the vector pointing from caster to target.

SimEvent::KnockbackApplied {
    tick, source_id, target_id,
    distance_x100,  // distance * 100 as integer
}

Pull

Pull { distance: f32 } drags the target toward the caster. The target's position is updated by calling move_towards(target_pos, caster_pos, distance), which moves the target along the vector pointing from target to caster.

SimEvent::PullApplied {
    tick, source_id, target_id,
}

Note: Displacement effects resolve instantly in a single tick. They do not check terrain walkability by default — a knocked-back unit can be pushed through walls. This matches the convention that displacement is a powerful effect with tactical consequences.

5. Swap

Swap is a parameterless effect that exchanges the positions of the caster and target. Both units instantly teleport to each other's former location.

// Resolution:
caster.position = target_pos;
target.position = caster_pos;

SimEvent::SwapPerformed {
    tick, unit_a: caster_id, unit_b: target_id,
}

Swap is a rare and powerful repositioning tool that can pull an enemy into your team or rescue an ally from a dangerous position.

6. Cover System

The cover_bonus field on UnitState represents damage reduction from nearby terrain. It ranges from 0.0 (no cover) to 0.5 (maximum cover, 50% damage reduction).

How Cover Is Calculated

The cover_factor() function in the pathing module evaluates cover based on the defender's position relative to both terrain and the attacker:

  1. Check the 8 cells adjacent to the defender for blocked (wall/obstacle) cells.
  2. For each blocked neighbor, compute the dot product between the wall direction and the attacker direction.
  3. If the wall lies between the defender and attacker (dot product > 0.3), it provides cover.
  4. Cardinal neighbors provide up to 0.5 cover; diagonal neighbors provide up to 0.3.

Cover is directional — a wall to your left provides no cover against an attacker on your right. The cover_bonus field is refreshed per-tick from the terrain context in the scenario/sim_bridge loop.

Fully open rooms have no blocked cells. When GridNav.fully_open = true, the cover calculation short-circuits to return 0.0, avoiding unnecessary per-cell iteration.

Cover also affects stealth detection: units with higher cover_bonus are harder to spot. See Crowd Control for stealth mechanics.

7. Elevation

The elevation field on UnitState tracks the unit's height above baseline in metres. It is stored per-cell in GridNav.elevation_by_cell and set via set_elevation_rect() during map generation.

Elevation Updates

Elevation is refreshed automatically after:

  • Standard movement via move_towards_position()
  • Fear-forced movement (fleeing in a random direction)

The terrain also defines a slope_cost_multiplier per cell (default 1.0). Cells on steep slopes have higher traversal cost during A* pathfinding, making units prefer flatter routes when available.

Elevation Advantage

The elevation_advantage() function returns the height difference between defender and attacker (positive = high ground). This value feeds into the terrain-biased movement system, where the AI scores candidate positions with an elevation bonus of elevation * 300 in the scoring function.

8. Movement-Restricting Effects

Several status effects prevent or limit a unit's ability to move. For full details on each CC type, see Crowd Control.

EffectMovement ImpactOther Restrictions
Root Cannot move. MoveTo intents are converted to Hold. Can still attack and cast abilities.
Stun Cannot act at all. The unit is fully locked — no movement, attacks, or abilities. Controlled via control_remaining_ms. Casting is cancelled.
Grounded Blocks dashes. Normal movement is unaffected, but non-blink dashes are prevented. Blink dashes (teleports) ignore Grounded.
Fear Forced flee. The unit runs away from the fear source at full move speed. All other actions are cancelled. Casting is cancelled. Movement direction is move_away(position, source_pos, max_delta).
Leash Range-limited. The unit is clamped to stay within max_range of an anchor point. It can move freely within the radius but is snapped back if it tries to leave. Applied via clamp_leash() after every MoveTo action.
Banish Removed from play. The unit cannot act, but is also untargetable. Casting is cancelled.
