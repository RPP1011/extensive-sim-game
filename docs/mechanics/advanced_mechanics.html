Advanced Mechanics

A comprehensive reference for the simulation's complex effect variants — stealth, tethers, damage sharing, temporal rewind, terrain creation, and more.

1. Stealth

The Stealth effect makes a unit untargetable by enemies for its duration. While stealthed, the unit cannot be selected as an attack or ability target. Stealth can be configured to break under specific conditions.

ParameterTypeDefaultDescription
duration_msu32requiredHow long stealth lasts, in milliseconds.
break_on_damageboolfalseIf true, stealth ends when the unit takes damage.
break_on_abilityboolfalseIf true, stealth ends when the unit casts an ability.

Implementation: a StatusKind::Stealth status effect is applied to the target. The targeting system checks for this status and skips stealthed units when resolving enemy targets. The AI can still issue Skulk and Hide intents to move tactically while stealthed. 

effect stealth {
  duration_ms = 3000
  break_on_damage = true
  break_on_ability = true
}

2. Leash

The Leash effect tethers a target to its current position. If the target moves beyond max_range, it is forced back within the leash radius. This is useful for zone-control abilities that prevent enemies from escaping.

ParameterTypeDefaultDescription
max_rangef32requiredMaximum distance the target can move from the anchor point.
duration_msu32requiredHow long the leash persists.

Implementation: a StatusKind::Leash status is applied with the target's current position stored as anchor_pos. Each tick, the simulation checks whether the unit has exceeded max_range from the anchor and pulls it back if so. 

effect leash {
  max_range = 5.0
  duration_ms = 4000
}

3. Link

The Link effect creates a damage-sharing bond between the caster and the target. When either linked unit takes damage, a percentage of that damage is shared with the other unit.

ParameterTypeDefaultDescription
duration_msu32requiredHow long the link persists.
share_percentf3250.0Percentage of damage transferred to the linked partner.

Implementation: both the caster and the target receive a StatusKind::Link status effect, each pointing to the other via partner_id. The damage pipeline checks for Link statuses and redirects share_percent of incoming damage to the partner.

Bidirectional. The link is symmetric — both units share damage with each other. This can be used defensively (link a tank to a squishy ally to spread damage) or offensively (link two enemies so damaging one hurts both).

effect link {
  duration_ms = 5000
  share_percent = 40
}

4. Redirect

The Redirect effect causes incoming damage to the target to be redirected to the caster instead. This acts as a protective bodyguard mechanic with a limited number of charges.

ParameterTypeDefaultDescription
duration_msu32requiredHow long the redirect persists.
chargesu323Number of damage instances that can be redirected before the effect expires.

Implementation: a StatusKind::Redirect status is applied to the protected target, storing the protector_id (the caster). When the target takes damage, the damage is routed to the protector instead, and one charge is consumed. The effect ends when all charges are used or the duration expires. 

effect redirect {
  duration_ms = 4000
  charges = 5
}

5. Rewind

The Rewind effect resets a unit to a previous position and HP value by reading from the unit's state_history ring buffer. This is a temporal mechanic that effectively "undoes" recent movement and damage.

ParameterTypeDefaultDescription
lookback_msu323000How far back in time to look, in milliseconds.

Implementation: the engine converts lookback_ms to a target tick (current_tick - lookback_ms / FIXED_TICK_MS) and searches the unit's state_history: VecDeque<(tick, position, hp)> ring buffer in reverse for the nearest entry at or before that tick. The unit's position and HP are restored (HP is capped at max_hp). A RewindApplied event is logged.

Ring buffer. The state_history field is populated automatically each tick by the simulation loop. It is marked #[serde(skip)] so it is not serialized — Rewind only works within a live simulation, not across save/load boundaries.

effect rewind {
  lookback_ms = 4000
}

6. CooldownModify

The CooldownModify effect adjusts ability cooldowns by a flat millisecond amount. It can reduce cooldowns (negative value) to let abilities come off cooldown faster, or increase them (positive value) to delay enemy abilities.

ParameterTypeDefaultDescription
amount_msi32requiredMilliseconds to add (positive) or remove (negative) from cooldowns. Negative values use saturating subtraction.
ability_nameOption<String>NoneIf specified, only affects the named ability. If omitted, affects all abilities on the target.

Implementation: the engine iterates over the target's abilities vector. If ability_name is set, only the matching slot is modified. For negative amount_ms, saturating_sub prevents underflow. For positive values, the cooldown is simply increased. 

// Reduce all cooldowns by 2 seconds
effect cooldown_modify {
  amount_ms = -2000
}

// Increase a specific ability's cooldown by 3 seconds
effect cooldown_modify {
  amount_ms = 3000
  ability_name = "Fireball"
}

7. Duel

The Duel effect forces two units into a 1v1 confrontation. Both the caster and the target receive a StatusKind::Duel status effect, each referencing the other as their partner_id. While dueling, units can only interact with their duel partner.

ParameterTypeDefaultDescription
duration_msu32requiredHow long the forced duel lasts.

Events

Two events are logged for each duel:

  • DuelStarted { tick, unit_a, unit_b } — emitted when the duel begins.
  • DuelEnded { tick, unit_a, unit_b } — emitted when the duel status expires or one participant dies.
effect duel {
  duration_ms = 5000
}

8. Obstacle

The Obstacle effect creates impassable terrain on the battlefield. Unlike most effects, obstacles are not applied to a target unit — they are handled at zone creation time and modify the pathfinding grid.

ParameterTypeDefaultDescription
widthf32requiredWidth of the obstacle in grid units.
heightf32requiredHeight of the obstacle in grid units.

Implementation: the apply_effect_extended function is a no-op for Obstacle — the actual terrain modification happens in the zone system, which populates blocked_cells on the ActiveZone and updates the GridNav pathfinding grid. Units cannot move through blocked cells. 

effect obstacle {
  width = 3.0
  height = 1.0
}

9. ProjectileBlock

The ProjectileBlock effect blocks incoming enemy projectiles for a duration. It creates a protective zone that intercepts projectiles before they reach their target.

ParameterTypeDefaultDescription
duration_msu32requiredHow long the projectile block lasts.

Implementation: currently applied as a StatusKind::Immunity status on the caster with immune_to: ["projectile"]. The projectile advancement system (advance_projectiles) checks for this immunity when resolving projectile collisions. 

effect projectile_block {
  duration_ms = 4000
}

10. Attach

The Attach effect causes the caster to become untargetable and physically attach to an allied unit, moving with them. This is used for "ride-along" abilities where a support character latches onto a carry.

ParameterTypeDefaultDescription
duration_msu320 (indefinite)How long the attachment lasts. A value of 0 means indefinite (internally treated as 60000ms).

Implementation: the caster receives two status effects — StatusKind::Attached { host_id } to track the host, and StatusKind::Banish to make them untargetable. The caster's position is snapped to the host's position immediately and continues to follow the host each tick. 

effect attach {
  duration_ms = 5000
}

11. EvolveAbility

The EvolveAbility effect permanently replaces one of the caster's abilities with its evolved variant. This is a one-time transformation — once evolved, the ability cannot revert.

ParameterTypeDefaultDescription
ability_indexusizerequiredIndex into the caster's abilities vector identifying which ability to evolve.

Implementation: the engine looks up the ability slot at ability_index and checks for an evolve_into field on the AbilityDef. If present, the entire ability definition is replaced with the evolved version, cooldown is reset to 0, and charges are refilled to the new definition's max_charges. An AbilityUsed event is logged with the prefix "Evolved: ". 

effect evolve_ability {
  ability_index = 0
}

12. Execute

The Execute effect instantly kills a target if its current HP is at or below a percentage threshold of its max HP. This is a finisher mechanic designed to close out low-health targets.

ParameterTypeDefaultDescription
hp_threshold_percentf32requiredHP percentage threshold. If the target's current HP / max HP * 100 is at or below this value, the target dies instantly.

Implementation: the engine computes the target's HP percentage. If it is at or below the threshold, the target's HP is set to 0, all dead-unit state is cleared, and two events are logged: ExecuteTriggered followed by UnitDied. Death triggers are then fired via fire_damage_triggers.

Order matters. Execute checks happen after other effects in the same ability resolve. If an ability deals damage and has an Execute effect, the damage lands first, potentially bringing the target below the threshold before Execute checks.

effect execute {
  hp_threshold_percent = 20
}

13. DeathMark

The DeathMark effect places a mark on the target that accumulates all damage taken during its duration. When the mark expires, it detonates, dealing a percentage of the accumulated damage as bonus damage.

ParameterTypeDefaultDescription
duration_msu32requiredHow long the mark lasts before detonating.
damage_percentf3250.0Percentage of accumulated damage dealt on detonation.

Implementation: a StatusKind::DeathMark status is applied with accumulated_damage: 0. The damage pipeline increments accumulated_damage whenever the marked unit takes damage. When the status expires, the engine deals accumulated_damage * damage_percent / 100 as bonus damage. 

effect death_mark {
  duration_ms = 3000
  damage_percent = 60
}

14. Reflect

The Reflect effect returns a percentage of incoming damage back to the attacker. This is a defensive buff that punishes enemies for attacking the protected unit.

ParameterTypeDefaultDescription
percentf32requiredPercentage of incoming damage reflected back to the attacker.
duration_msu32requiredHow long the reflect buff lasts.

Implementation: a StatusKind::Reflect status is applied. When the damage pipeline processes an incoming hit, it checks for Reflect statuses on the target. If present, damage * percent / 100 is dealt back to the attacker as reflected damage. 

effect reflect {
  percent = 30
  duration_ms = 5000
}

15. Lifesteal

The Lifesteal effect heals the attacker for a percentage of damage dealt. This is a sustain buff that rewards aggressive play by converting outgoing damage into healing.

ParameterTypeDefaultDescription
percentf32requiredPercentage of damage dealt that is converted to healing for the attacker.
duration_msu32requiredHow long the lifesteal buff lasts.

Implementation: a StatusKind::Lifesteal status is applied. The damage pipeline checks for Lifesteal statuses on the attacking unit. When present, the attacker is healed for damage_dealt * percent / 100 after applying damage to the target. 

effect lifesteal {
  percent = 25
  duration_ms = 6000
}
