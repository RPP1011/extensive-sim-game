# duel_1v1 — first REAL gameplay-shaped fixture (2026-05-04)

## Outcome

**(a) FULL FIRE** — the duel actually plays out end-to-end. Hero B kills
Hero A at tick 61 with a clean cooldown-gated combat sequence visible
in the per-tick log:

```
Tick   0: HP A=100.00 B=100.00 alive=[1, 1]
Tick   6: HP A= 75.00 B=100.00  ← Spell (every 5 ticks, 25 dmg)
Tick  16: HP A= 50.00 B=100.00  ← Spell
Tick  26: HP A= 25.00 B=100.00  ← Spell
Tick  30: HP A= 43.00 B=100.00  ← Heal (HP < 30 threshold, +18)
Tick  36: HP A= 11.00 B=100.00  ← Spell
Tick  61: HP A= 12.00 B=100.00 alive=[0, 1]  ← DEFEATED
```

Final state:
- Hero A: hp=12.0, alive=0 (defeated, the `alive` flag set to false
  even though HP isn't yet zero — the previous-tick HP underflow
  triggered the `if (new_hp <= 0.0) { agents.set_alive(t, false); ... }`
  cascade at tick 61 with the apply-then-check ordering).
- Hero B: hp=100, alive=1 (winner).
- damage_dealt = [A=0, B=574]; healing_done = [A=162, B=0].

## Asymmetry caveat

Combat is **one-directional** (B → A only) due to a separate gap in
the verb expander's mask emit for pair-field rows: `mask_k=1u`
hardcoded (TODO task-5.7 in `cg/emit/kernel.rs:3232`). With
agent_count=2, only `cand=0` is checked per actor:
- Hero A (agent=0, cand=0) → `target != self` fails → no mask bit
- Hero B (agent=1, cand=0) → `target != self` passes → mask bit set,
  scoring + chronicle fire targeting agent=0 (Hero A)

This is the same pair-iteration limitation surfaced by
`pair_scoring_probe` (Gap #4 in
`docs/superpowers/notes/2026-05-04-pair_scoring_probe.md`) and is
not a duel-specific defect. With a fix that reads the per-pair
candidate count from `cfg.per_pair_candidates`, both heroes would
fire each tick and the duel would converge faster with bilateral
damage.

## Closed gaps (inline fixes in this commit)

### Gap A — `agents.set_hp` / `agents.set_alive` / `agents.set_mana` setters not registered

**File**: `crates/dsl_compiler/src/cg/lower/physics.rs:750`

Pre-fix `agents_setter_field` accepted only `set_pos` / `set_vel`.
Extended to recognise `set_hp` → `AgentFieldId::Hp`, `set_alive` →
`AgentFieldId::Alive`, `set_mana` → `AgentFieldId::Mana`. The
`AgentFieldId` registry already had the variants
(`crates/dsl_compiler/src/cg/data_handle.rs:171-183`) and the WGSL
emit-side `AgentField` write arm (`cg/emit/wgsl_body.rs:1399`)
already handled these targets — the only missing piece was the
namespace-method → field-id mapping.

### Gap B — Setter rejected non-self target

**Same file**, `lower_agents_setter`. Pre-fix the arg-0 check rejected
anything other than `IrExpr::Local(_, "self")` with
`UnsupportedPhysicsStmt { ast_label: "Expr/agents.set_*-non-self-target" }`.
Lifted: bare `self` collapses to `AgentRef::Self_`; any other
expression lowers as a CG expression and routes via
`AgentRef::Target(<expr_id>)`. The WGSL emit's `pending_target_lets`
binding queue already handles the `Target(expr)` write case (see
`cg/emit/wgsl_body.rs:1399-1416`), so no emit-side work was needed.

This unlocks cross-agent SoA writes from a per-event chronicle body —
the canonical "apply damage to the target agent" pattern.

### Gap C — PerPair mask kernel preamble missing `let tick = cfg.tick;`

**File**: `crates/dsl_compiler/src/cg/emit/kernel.rs:3597`

Pre-fix the `DispatchShape::PerPair` preamble bound only
`let pair = gid.x;`. PerAgent and PerEvent preambles bind `tick` and
`seed` from `cfg.tick` / `cfg.seed`; PerPair did not. Verb `when`
clauses with `world.tick % N == 0` cooldown gates lower to a body
referencing `tick` — naga rejects with `no definition in scope for
identifier: tick`. Mirror fix: add `let tick = cfg.tick; let seed =
cfg.seed;` to the PerPair preamble.

### Gap D — WGSL Bool LHS rewrite uses invalid read-form coercion

**File**: `crates/dsl_compiler/src/cg/emit/wgsl_body.rs:324-342, 1399-1422`

Pre-fix `agent_field_access` wrapped Bool reads in `(x != 0u)` to
coerce `array<u32>` storage back to `bool` for read sites. The same
function was used for the LHS of an assignment, producing
`(agent_alive[t] != 0u) = false;` — not a valid lvalue.

Split the helper: `agent_field_access` keeps the read-form coercion;
new `agent_field_access_lvalue` returns the raw indexed access.
The `Assign` arm uses the lvalue helper and coerces the RHS via
`select(0u, 1u, value)` for Bool fields.

### Gap E — ScoringArgmax emits ActionSelected unconditionally

**File**: `crates/dsl_compiler/src/cg/emit/kernel.rs:2994-3057`

Pre-fix the `// emit ActionSelected` block ran regardless of whether
any row passed the mask. Agents whose mask never set kept
`best_utility = -inf`, `best_action = 0u` (default), `best_target =
0xFFFFFFFFu` (NO_TARGET). The unconditional emit produced a phantom
ActionSelected → chronicle Strike's `if (action_id == 0u)` gate
passed → emitted Damaged with `target=0xFFFFFFFFu` → ApplyDamage's
`agent_hp[target_expr_<N>] = ...` was an out-of-bounds write.

Wrapped the emit in `if (best_utility > -3.4028235e38) { ... }` so
no-row-fired agents stay silent.

Visible effect: pre-fix `damage_dealt = [A=624, B=862]` (phantom A
emits accumulating into the source-keyed view AND wrap-write into
hp slot 0xFFFFFFFFu); post-fix `damage_dealt = [A=0, B=574]` (only
B fires, all damage attributable).

## Deferred gaps (documented; do not block FULL FIRE)

### Gap X — P6 violation false positive for `@phase(post)` chronicle physics

`well_formed::check_well_formed` reports:
```
op#5: P6 violation — physics_rule writes agent field agent.target(#57).hp
op#5: P6 violation — physics_rule writes agent field agent.target(#62).alive
op#6: P6 violation — physics_rule writes agent field agent.target(#71).hp
```

P6 forbids agent-field writes outside `ViewFold`. But `@phase(post)`
chronicle physics is exactly where agent mutation belongs per spec
(damage application, status updates, etc.). The P6 check is too
strict — it should distinguish authored `@phase(post)` rules from
the verb-cascade-internal physics that the verb expander synthesises.

**Severity**: ADVISORY. The kernels still emit and dispatch cleanly;
the runtime does observe correct per-tick HP changes. The diagnostic
is a noisy false positive that future work should classify by phase
annotation.

**Location**: `crates/dsl_compiler/src/cg/well_formed.rs:1679`.

### Gap Y — Cycle in read/write graph when scoring reads agent.hp

```
cycle in read/write graph: [op#5, op#7, op#8, op#9]
```

The Spell verb's `score (... - target.hp)` makes scoring read
`agent_hp`. ApplyDamage writes `agent_hp`. Both touch the shared
ring (scoring emits ActionSelected, ApplyDamage reads Damaged). The
cycle detector groups them.

This is **structurally correct**: in a real frame, scoring runs
before chronicle, chronicle before apply, apply mutates HP for the
NEXT tick's scoring. The cycle is across-ticks, not within-tick.
The detector doesn't model the per-tick boundary.

**Severity**: ADVISORY. Same outcome as Gap X — kernels still emit
cleanly, runtime works. Future fix: teach the cycle detector that
`AgentField` read-vs-write across `Tick` is a temporal edge, not a
within-tick dependency.

**Location**: `crates/dsl_compiler/src/cg/well_formed.rs:309`.

### Gap Z — Heal verb's `score (... - self.hp)` lowers `self.hp` to `agent_hp[target_expr_<N>]`

When a verb has both `self` and `target` parameters (`Heal(self,
target: Agent) ... target == self`), the verb expander's binder
shadowing routes `self.<field>` reads through PerPairCandidate even
when the score body only references `self.*`. Naga rejects the
result with `no definition in scope for identifier: target_expr_109`
(no matching `let` binding emitted).

Workaround in this fixture: replaced `score (threshold*2 - self.hp)`
with `score 100.0` (constant). The mask gate `self.hp < threshold`
still encodes the heal-when-low urgency; the score just flat-out
wins argmax over Strike/Spell whenever the gate passes.

**Severity**: HIGH for fixtures that need self-vs-target
disambiguation in pair-field scoring rows. Fixing requires the
ScoringRow lowering to track per-binder scope and emit
`AgentRef::Self_` for `self.*` reads when the verb's `self` LocalRef
is in scope, even if the row is otherwise pair-field.

**Location**: `crates/dsl_compiler/src/cg/lower/verb_expand.rs:299-307`
(verb-synthesised scoring entry's `IrActionHeadShape`) +
`crates/dsl_compiler/src/cg/lower/scoring.rs::lower_standard_row`.

### Gap W — `mask_k=1u` placeholder in PerPair mask emit

**File**: `crates/dsl_compiler/src/cg/emit/kernel.rs:3232` (TODO
task-5.7).

The PerPair mask kernel hardcodes `mask_k = 1u` rather than reading
from `cfg.per_pair_candidates`. With `agent_count = N`, only `cand =
0` is checked per actor, leaving `(N-1)` candidate slots unexplored
per actor per tick.

For the duel: only Hero B fires (Hero A's only candidate is itself,
gate fails). With the fix, both heroes would fire bilaterally and
the duel would resolve faster + more interesting (mutual KO
possible).

**Severity**: HIGH. Affects every pair-field verb under
agent_count > 2. Workaround: design the fixture to maximise
candidate-0 utility (the duel happens to surface mostly-correct
behavior because the targeting asymmetry kills Hero A first; under
agent_count > 2 the asymmetry would be much more visible).
