# Emitted WGSL Body Fill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace every stub WGSL body in `crates/engine_gpu_rules/src/*.wgsl` with a real one — either by wiring xtask to invoke an existing `dsl_compiler` emitter, or by recovering the hand-written WGSL from git history (`4474566c~1`) and adapting its binding layout to match the post-T16 emitted Rust kernel side. **Every emitted WGSL must pass naga parse-validation as the gate** — `include_str!` only catches Rust-side compile errors, not WGSL-side naga rejections.

## Architectural reality discovered 2026-04-28

The first attempt at Task 1 (wire `emit_physics_wgsl`) revealed: **emitters are not all self-contained.** Two model families exist among the existing emitters:

- **Self-contained emitters** (verified working today): `emit_masks_wgsl_fused`, `emit_scoring_wgsl_atomic_views`. They embed all WGSL helpers inline (`alive_bit`, view-read wrappers, etc.). Output is naga-parseable as-is.

- **Prelude-dependent emitters** (verified broken today): `emit_physics_wgsl`. The output references undefined symbols (`events_in`/`EventSlot` typed buffer, `state_agent_*` getters/setters, `gpu_emit_event`, `EVENT_KIND_*` consts, `slot_of`, `alive_bit`, `wgsl_world_tick`, `abilities_is_known`, `abilities_effects_count`, view-read fns) that pre-T16 came from hand-written WGSL constants in the deleted kernel files. Without those, naga rejects the shader.

The other emitters (`emit_pick_ability_wgsl`, `emit_view_fold_wgsl`, `emit_per_entity_ring_fold_wgsl`, `emit_symmetric_pair_topk_fold_wgsl`) are unverified — they may be self-contained or prelude-dependent. **Each must be tested individually before declaring its task complete.**

The Bucket-R kernels (recovered hand-written WGSL) likely embed their helpers inline by construction (the old hand-written constants were self-contained shaders), but the binding-layout adaptation may break that — verify per kernel.

## Strategy revision

**Task 0 (NEW — gating infrastructure):** Add a naga-parse test in `engine_gpu_rules` that asserts every `*.wgsl` file under `engine_gpu_rules/src/` parses cleanly as WGSL via `naga::front::wgsl::parse_str`. With current stub bodies, this passes (stubs are valid). After each subsequent task, the test must continue to pass. **This is the gate that catches the runtime-prelude gap at test time, not at first-device-instantiation.**

**Task 1 (REVISED):** Wire `emit_physics_wgsl` AND ensure naga accepts the result. Since the emitter is prelude-dependent, this task includes producing the runtime prelude — likely a new `dsl_compiler::emit_physics_runtime_wgsl()` that defines every state accessor in terms of the standard BGL. The xtask emit step concatenates `prelude + emitted physics body` into `physics.wgsl`. Naga-parse test passes ⇒ task done.

**Tasks 2-7:** For each emitter, run the naga-parse test as the gate. If self-contained, just wire. If prelude-dependent, produce the matching runtime prelude (often shared across multiple kernels — physics + pick_ability may share state accessors).

**Task 8 (unchanged):** Final verification + closeout.

## Why this is harder than originally framed

The pre-T16 hand-written WGSL constants were ~1000-3000 lines each. Most of that volume was the runtime prelude (state accessors, event push primitives, ability registry walking). The `dsl_compiler` emitters only produce the rule logic part (~400 lines). Reconstructing the prelude as a `dsl_compiler` module is real engineering — likely 500-1500 lines of WGSL infrastructure shared across emitters.

The honest scope estimate has grown from "8 tasks" to "8-12 tasks plus a runtime-prelude design subtask," with high judgment cost on the prelude design.

- **P10 (no runtime panic):** WGSL kernels can't panic (no `unwrap` etc.); divergent control flow on GPU is bounded. Host-side: this stream adds no new panic paths.
- **P11 (reduction determinism):** atomic-append ordering is already encoded in the per-tick `seq` injection (T11 work) and `sort_by(target_id)` for materialized views. This stream restores the WGSL bodies that consume those — Stream C verifies byte-equality.

## Critical pre-flight: Binding-layout audit

The recovered hand-written WGSL was written against the OLD binding layouts (typed `array<AgentSlot>`, etc.). The post-T16 emitted Rust side uses the NEW binding layouts (often `array<u32>` with helper struct decoding). For every Bucket R kernel, the recovered WGSL body must be ADAPTED to consume the new binding layout, not pasted verbatim.

The audit recipe (used in every Bucket R task):
1. Read the current `crates/engine_gpu_rules/src/<kernel>.wgsl` to see the new binding layout.
2. Run `git show 4474566c~1:crates/engine_gpu/src/<source>.rs | grep -A 60 'WGSL.*r#"'` to recover the old WGSL.
3. Diff the binding declarations. Where the new binding is `array<u32>`, the old WGSL's typed `array<AgentSlot>` reads must be rewritten as offset-indexed `array<u32>` reads using the `GpuAgentSlot` field offsets defined in `crates/engine_gpu/src/sync_helpers.rs::GpuAgentSlot` (16 fields × 4 bytes each = 64-byte stride).
4. Replicate the kernel logic with the new binding layout.
5. Verify naga accepts it (`cargo build -p engine_gpu_rules --features gpu`).

If the binding layouts diverge so radically that 1:1 adaptation isn't viable, that's a signal the kernel needs a fresh emitter design rather than a recovery — surface it to the controller.

```rust
//! Asserts every emitted WGSL file under engine_gpu_rules/src/ parses
//! cleanly as WGSL. Catches missing-symbol errors at test time instead
//! of at first device-instantiation. megakernel.wgsl is excluded — it's
//! intentional T14 scaffold owned by gpu_megakernel_plan.

use std::path::PathBuf;

#[test]
fn every_wgsl_parses() {
    let src_dir: PathBuf = [env!("CARGO_MANIFEST_DIR"), "src"].iter().collect();
    let mut failures: Vec<(String, String)> = Vec::new();

    for entry in std::fs::read_dir(&src_dir).expect("read src dir") {
        let entry = entry.expect("entry");
        let path = entry.path();
        let Some(ext) = path.extension() else { continue };
        if ext != "wgsl" { continue; }
        let name = path.file_name().unwrap().to_string_lossy().to_string();
        if name == "megakernel.wgsl" { continue; }  // T14 scaffold

        let src = std::fs::read_to_string(&path).expect("read wgsl");
        match naga::front::wgsl::parse_str(&src) {
            Ok(_) => {}
            Err(e) => failures.push((name, format!("{e:#}"))),
        }
    }

    if !failures.is_empty() {
        let mut msg = String::from("naga parse failures:\n");
        for (name, err) in &failures {
            msg.push_str(&format!("  {name}:\n{err}\n\n"));
        }
        panic!("{msg}");
    }
}
```

```rust
//! WGSL emitter for AlivePackKernel — packs the alive bit per agent
//! into a u32 bitmap. Recovered from pre-T16 hand-written constant
//! and adapted to the post-T16 emitted Rust binding layout.

pub fn emit_alive_pack_wgsl() -> String {
    String::from(r#"
@group(0) @binding(0) var<storage, read>       agents:       array<u32>;
@group(0) @binding(1) var<storage, read_write> alive_bitmap: array<u32>;
struct AlivePackCfg { agent_cap: u32, _pad0: u32, _pad1: u32, _pad2: u32 };
@group(0) @binding(2) var<uniform>             cfg:          AlivePackCfg;

const SLOT_STRIDE: u32 = 16u;       // 16 × u32 per GpuAgentSlot
const ALIVE_OFFSET: u32 = 4u;       // pos[3] + hp + max_hp + shield + attack ... etc — adjust per audit

@compute @workgroup_size(64)
fn cs_alive_pack(@builtin(global_invocation_id) gid: vec3<u32>) {
    let i = gid.x;
    if (i >= cfg.agent_cap) { return; }
    let alive_word = i / 32u;
    let alive_bit  = 1u << (i % 32u);
    let alive = agents[i * SLOT_STRIDE + ALIVE_OFFSET];
    if (alive != 0u) {
        atomicOr(&alive_bitmap[alive_word], alive_bit);  // see note re: storage atomics — coordinate w/ binding type
    }
}
"#)
}
```

(The above is illustrative — Step 3's audit determines the exact constants and layout. The `atomicOr` may need binding-type adjustment to `array<atomic<u32>>` — verify with naga.)
