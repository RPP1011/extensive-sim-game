What Already Exists

The first three weeks of development focused on the simulation core and AI infrastructure. Every system below is implemented, tested, and integrated:

✓
Deterministic Combat Simulation
Pure functional step(state, intents, dt_ms) at 100ms fixed tick. All randomness flows through seeded RNG. Determinism verified in CI across debug and release builds.
✓
Hero System
TOML stat definitions + ability DSL. 27 original heroes, 172 LoL champion imports. Hybrid format is the source of truth for all hero content.
✓
Effect System
Five composable dimensions: Effect (what), Area (where), Delivery (how), Trigger (when), Tags (power). Plain data structs dispatched via pattern matching.
✓
AI Decision Pipeline
Five-layer intent generation: Squad AI → Ability Evaluator → Transformer Decision Head → GOAP Planner → Control AI. Each layer can override.
✓
IMPALA V4 Self-Play RL
4,096 parallel sims, V-Trace off-policy correction, behavioral embeddings via ability_operator crate. REINFORCE + PPO policy learning.
✓
Burn GPU Inference
Migrated from Python sidecar to native Rust via Burn framework. Single model definition for CPU (tests) and CUDA (training). 1.9x speedup, zero Python processes.
✓
Room Generation
Procedural floorplan generation with cover placement, navigation mesh, and ML-assisted Diffusion Transformer (DiT) for layout variety.
✓
Campaign & Persistence
Campaign types with save/load, Voronoi-based overworld map generation, faction territories, and flashpoint systems.
✓
Scenario Runner & sim_bridge
TOML-based scenario configuration, headless sim execution, NDJSON protocol for external agent integration, coverage-driven scenario generation.
✓
DSL Parsers
winnow-based parsers for both .ability and .goap file formats. AST → lowering → runtime data structures.

Foundation status: The simulation engine, content pipeline, AI decision system, and training infrastructure are all operational. What follows is the plan to build the game around these systems.

Architecture

The game is a three-layer simulation with two cross-cutting AI systems. The LFM (Language Foundation Model) generates all narrative content, while the RL Agent playtests everything the LFM produces. The layers feed data upward; the AI systems cut across all three.

Interactive: System Architecture

The ASCII UI Vision

The entire game renders as ASCII art through egui with full RGBA color per character cell. No sprites, no textures — every pixel is a colored glyph. The glyph atlas pre-rasterizes characters so rendering is just texture lookups.

Seven Screens

1. Start Menu
Campaign selection · New / Load / Seed import

ASCII art title with animated effects. Campaign seed input for sharing. Browse existing saves or generate a new world from a seed. Model tier selection (Micro/Standard/Full) based on detected hardware.

2. Character Creation
Party assembly · Culture selection · Trait allocation

Select from available hero archetypes. Assign culture and personality traits that affect morale modifiers. ASCII portrait preview generated by the ASCII art model. Party composition analysis showing synergies and weaknesses.

3. Overworld
150x80+ ASCII terrain · Faction territories · Roaming parties

Procedural terrain rendered as colored ASCII: ^ mountains, ~ water, . plains, T forests. Voronoi region boundaries with faction color tints. 30+ AI-controlled parties roaming with visible intent indicators. Road networks connecting settlements. Scrollable viewport with minimap overlay.

4. Region View
Zoomed region · Points of interest · Local events

Zoomed view of a single Voronoi region showing individual locations: towns, dungeons, camps, ruins. Local faction influence meters. Active quest markers. NPC interaction points with LFM-generated dialogue.

5. Eagle Eye
Strategic overlay · Faction relations · Global state

Full-map strategic view showing faction territories, alliance networks, trade routes, and conflict fronts. Morale heatmap overlay showing which regions are stable vs volatile. Timeline of recent events.

6. Mission Execution
Multi-room dungeon · Concurrent combat windows · Real-time tactics

OS-style window manager with draggable, resizable combat viewports. Multiple simultaneous battles when the party splits. Each combat window shows the deterministic 100ms-tick simulation with unit positions, ability cooldowns, health/morale bars. Cover indicators from procedurally generated rooms.

7. Replay
Deterministic playback · Speed control · AI decision overlay

Because the simulation is deterministic, any combat can be replayed from its seed + initial state. Variable speed playback (0.25x–8x). AI decision overlay showing what each unit’s AI pipeline chose and why — which layer overrode, what the transformer attended to, GOAP plan visualization.

Morale & Culture in the UI

The morale system is visible everywhere. In combat, each unit has a morale bar alongside health. Culture traits appear as colored glyphs next to unit names. When morale cascades trigger — a unit routes, adjacent allies waver, the healer panics — the ASCII art conveys this through color shifts: confident units glow bright, wavering units dim, routing units flash red.

On the overworld, faction morale manifests as territory color saturation. Stable factions have deep, rich tints; fractured factions show pale, flickering borders. You can see which factions are about to collapse before the war even starts.

LFM Architecture

The Language Foundation Model is a bundled, finetuned language model that generates all narrative content in the game. No API calls, no internet required — the model ships with the game and runs inference locally via Burn. The key insight: different content has different latency requirements, so we split generation into three tiers.

Three Content Tiers

Tier When Generated Latency Budget Examples
Static Compile time / hand-authored None Ability definitions, stat formulas, simulation rules, core mechanics. The sim-critical data that must be deterministic and exact.
AOT (Ahead-of-Time) Campaign creation (loading screen) Minutes Faction histories, character backstories, quest lines, dialogue trees, location descriptions, name generators. Generated once per campaign seed, cached permanently.
Runtime During gameplay < 2 seconds Reactive dialogue, battle commentary, dynamic event descriptions, NPC responses to unexpected player actions.

Seeded Determinism

The entire AOT pipeline is seeded. The same campaign seed on the same model tier produces identical content, regardless of hardware. This enables:

  • Campaign seed sharing — share a seed string and your friend gets the exact same world
  • Reproducible bug reports — “seed X at stage Y produces broken content”
  • RL agent training — the playtester can replay the same world thousands of times
  • Content regression testing — after model updates, verify key seeds still produce quality content

The Compounding Loops

The real value of this architecture isn’t any single system — it’s how they compound. Each system makes the others better in a self-reinforcing cycle.

Interactive: Feedback Loops

Loop 1: RL ⇄ LFM

The primary feedback loop. The LFM generates campaign content (factions, quests, dialogue). The RL playtester agent plays through that content thousands of times, collecting quality-diversity metrics: win rates, strategy diversity, content coverage, degenerate pattern frequency. These metrics feed back into LFM finetuning — the model learns which content patterns produce interesting gameplay.

Generate → Playtest → Evaluate → Improve → Generate. Each cycle produces better content and a better playtester. The playtester learns new strategies from novel content; the content generator learns what makes the playtester struggle productively.

Loop 2: Combat Model Retraining

The existing IMPALA combat policy was trained on hand-authored heroes and abilities. When the LFM generates new content — custom abilities, novel faction matchups, unusual terrain configurations — the combat model needs to adapt. The RL playtester automatically generates training data from LFM content, and the combat model retrains on this expanded dataset. More content diversity → more robust combat AI.

Loop 3: Community Content

The content schema system means community modders can create new factions, abilities, and quests using the same TOML/DSL format. This content feeds back into LFM training data, expanding the model’s understanding of what “good content” looks like. The RL playtester validates community content before it ships — automated QA at scale.

Loop 4: Seed Sharing

Campaign seeds become a social feature. Players share seeds that produce particularly interesting worlds. Popular seeds get more playtester attention, their quality metrics feed into the LFM, and the model learns to generate worlds with similar properties. The community curates the training data through play.

Research Publication Opportunities

Several components of this system are novel enough to warrant academic publication. Each builds on the project’s unique combination of deterministic simulation, hierarchical RL, and integrated language model generation.

Hierarchical RL Playtesting with Quality-Diversity Objectives
Venues: NeurIPS, ICML, AAAI
An RL agent that operates at three timescales (strategic/tactical/micro) to playtest generated game content. Quality-diversity metrics replace simple win-rate optimization. The agent discovers degenerate strategies, unwinnable scenarios, and content gaps that human testers miss.
LFM-in-the-Loop Procedural Content Generation with RL Validation
Venues: IEEE CoG, FDG, AIIDE
A language model generates structured game content that an RL agent validates through play. The feedback loop improves both the generator and the validator simultaneously. First demonstration of closed-loop LLM + RL content generation for complex strategy games.
Culture-Weighted Morale Cascades in Multi-Agent Tactical Combat
Venues: AAMAS, IEEE CoG
Morale as a first-class simulation mechanic with cultural modifiers. Shows that culture-weighted morale cascades produce emergent tactical behaviors (sacrificial charges, fighting withdrawals, last stands) that emerge naturally from simple per-agent morale rules.
Seeded Deterministic Content Generation for Reproducible Game Worlds
Venues: FDG, PCG Workshop, AAAI
A technique for making language model generation deterministic across hardware tiers via seeded sampling and cached intermediate states. Enables campaign seed sharing, reproducible bug reports, and regression testing of generated content.
