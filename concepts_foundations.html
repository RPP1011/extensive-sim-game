Neural Networks

A neural network is a function built from simple building blocks: layers of numbers (neurons), connected by weights. Each layer takes a vector of numbers, multiplies by a weight matrix, adds a bias, and passes the result through a nonlinear activation function. Stacking layers lets the network learn arbitrarily complex patterns.

Key terms:

  • Neuron — A single number in a layer. Computes: output = activation(w·x + b).
  • Weight (w) — A learnable number that scales an input. The network “learns” by adjusting weights.
  • Bias (b) — A learnable offset added after multiplication. Lets the neuron shift its output.
  • Linear layer (Dense/Fully-connected) — y = Wx + b. Every input connects to every output. The workhorse of neural nets.
  • Activation function — A nonlinear function applied after the linear transform. Without it, stacking layers would just be one big matrix multiply. Common: ReLU (max(0,x)), tanh, sigmoid.
  • ReLU — Rectified Linear Unit: f(x) = max(0, x). Dead simple, works great. Negative values become 0, positive pass through.
  • Parameters — All the weights and biases combined. “A 113K-parameter model” means 113,000 learnable numbers.
  • Forward pass — Running an input through the network to get an output. No learning happens here — just computation.

Gradient Descent

Gradient descent is how neural networks learn. The gradient of the loss tells you which direction to adjust each weight to reduce error. Take a small step in that direction. Repeat thousands of times. The network “rolls downhill” on the loss landscape until it finds a good solution.

θ ← θ − α · ∇L(θ)

Key terms:

  • Gradient (∇L) — A vector of partial derivatives. Points in the direction of steepest increase of the loss. We go the opposite way.
  • Learning rate (α) — Step size. Too big = overshoot and diverge. Too small = takes forever. Typical: 0.001–0.0001.
  • Backpropagation — The algorithm that efficiently computes gradients by working backwards through the network, applying the chain rule layer by layer.
  • Chain rule — If y = f(g(x)), then dy/dx = f'(g(x)) · g'(x). This is how gradients flow through stacked layers.
  • SGD (Stochastic Gradient Descent) — Compute the gradient on a random batch instead of the entire dataset. Noisy but fast.
  • Adam / AdamW — Optimizers that adapt the learning rate per-weight using momentum and variance estimates. AdamW adds weight decay (regularization).
  • Convergence — When the loss stops decreasing meaningfully. The model has “learned” what it can.

Loss Functions

A loss function measures how wrong the model's prediction is. The entire point of training is to minimize this number. Different tasks need different losses: regression uses MSE, classification uses cross-entropy. The loss is the signal that gradient descent follows downhill.

Key terms:

  • MSE (Mean Squared Error) — Average of (prediction − target)². Used for continuous values (HP prediction, position). Penalizes big errors heavily.
  • Cross-Entropy — −∑ y·log(p). Used for classification (which action to take). Measures how surprised the model is by the correct answer.
  • L1 / MAE — Average of |prediction − target|. More robust to outliers than MSE.
  • Regularization — Extra penalty added to the loss to prevent overfitting. L2 (weight decay): penalizes large weights. Keeps the model simple.
  • Training loss vs Validation loss — Training loss is on data the model learns from. Validation loss is on held-out data. When they diverge, the model is overfitting.

Batch, Epoch & Learning Rate

Training a neural network means showing it data in batches, updating weights after each batch. One full pass through the dataset is an epoch. The learning rate controls how aggressively the model updates. These three knobs control the training loop.

Key terms:

  • Batch — A subset of the training data processed together. Batch size 512 means 512 examples per gradient step. Larger batches = smoother gradients but more memory.
  • Epoch — One complete pass through the entire training dataset. Training typically runs for many epochs (10–1000+).
  • Iteration / Step — One weight update. Steps per epoch = dataset_size / batch_size.
  • Learning rate schedule — Changing the learning rate during training. Common: start high (explore), decay over time (fine-tune). Warmup: start low, ramp up.
  • Convergence — When loss plateaus. More epochs won't help — the model has learned what it can from this data with this architecture.

Overfitting & Generalization

Overfitting is when a model memorizes the training data instead of learning general patterns. It gets a perfect score on data it has seen, but fails on new data. Generalization is the goal: a model that performs well on unseen data. The tension between the two is the central challenge of machine learning.

Key terms:

  • Training set — Data the model learns from. It sees this data repeatedly.
  • Validation set — Held-out data used to check generalization during training. Never trained on.
  • Test set — Final evaluation data. Used once to report results.
  • Underfitting — Model is too simple to capture the pattern. Both train and val loss are high.
  • Overfitting — Model is too complex for the data. Train loss is low, val loss rises.
  • Dropout — Randomly zero out neurons during training. Forces the network to not rely on any single neuron. A form of regularization.
  • Early stopping — Stop training when validation loss starts rising, even if training loss is still falling.
  • Model capacity — How many parameters / how complex the model is. More capacity = more risk of overfitting, but can learn harder patterns.

Normalization

Normalization rescales data or activations to a standard range. Without it, different features with wildly different scales (HP: 0–5000, position: 0–30) would make training unstable. Inside networks, normalization between layers keeps the signal from exploding or vanishing as it flows through.

Key terms:

  • Layer Normalization (LayerNorm) — Normalizes across features within a single example. Used in transformers. Makes each layer's input have mean=0, std=1.
  • Batch Normalization (BatchNorm) — Normalizes across the batch dimension. Popular in CNNs. Depends on batch statistics, so behaves differently at train vs test time.
  • L2 Normalization — Scales a vector so its length (L2 norm) = 1. Used for embeddings so similarity = dot product = cosine similarity.
  • Feature scaling — Rescaling input features to [0,1] or mean=0/std=1 before feeding to the network. Critical for stable training.
  • Pre-norm vs Post-norm — Whether normalization happens before or after the attention/FF block. Pre-norm (used in our model) trains more stably.
