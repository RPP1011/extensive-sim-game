REINFORCE

REINFORCE (Williams, 1992) is the simplest policy gradient algorithm. The idea: run your policy, see what happens, and increase the probability of actions that led to good outcomes. No value function, no critic — just the raw signal from episode returns.

∇J(θ) = E[∑t ∇ log πθ(at|st) · Gt]

Key terms:

  • πθ(a|s) — The policy: probability of taking action a in state s, parameterized by neural network weights θ.
  • Gt — The return: discounted sum of future rewards from time t. Gt = rt + γrt+1 + γ²rt+2 + …
  • γ (gamma) — Discount factor (0–1). Lower values make the agent more myopic.
  • log π — Log probability of the chosen action. Gradient of this is the score function.
  • Baseline — Subtract mean return to reduce variance: Gt − b. Doesn't change the expected gradient, but dramatically reduces noise.

PPO (Proximal Policy Optimization)

PPO (Schulman et al., 2017) fixes REINFORCE's instability by limiting how much the policy can change in each update. Instead of a raw gradient step, PPO computes an importance-weighted ratio between old and new policies, then clips it to prevent catastrophic updates.

LCLIP = E[min(rt(θ) Ât, clip(rt(θ), 1−ε, 1+ε) Ât)]

Key terms:

  • rt(θ) = πθ(a|s) / πθold(a|s) — The probability ratio. r=1 means the policy hasn't changed; r=2 means the action is now twice as likely.
  • ε (epsilon) — Clip range, typically 0.2. Limits the ratio to [0.8, 1.2], preventing large policy jumps.
  • Ât (advantage) — How much better this action was than expected. Computed via GAE (Generalized Advantage Estimation).
  • GAE(γ, λ) — Smooths advantage estimates over multiple timesteps. λ=0 is one-step TD, λ=1 is Monte Carlo.
  • Clipping — The min + clip removes the incentive for the ratio to go outside [1−ε, 1+ε]. This is what makes PPO stable.
Clip ε: 0.20    Advantage: +1.0

IMPALA & V-Trace

IMPALA (Espeholt et al., 2018) is a distributed actor-critic architecture. Many actors collect experience in parallel using a slightly stale policy, while a central learner updates the weights on GPU. V-Trace corrects for the mismatch between the actor's behavior policy and the learner's target policy using importance sampling with truncated weights.

vs = V(xs) + ∑t=ss+n-1 γt-s (∏i=st-1 ci) δtV

Key terms:

  • Actor — A process running the simulation with a (possibly stale) copy of the policy. Produces trajectories.
  • Learner — The GPU process that consumes trajectories and updates the policy. Runs asynchronously from actors.
  • Off-policy — The actor's policy (μ) differs from the learner's policy (π) because the weights were updated since the trajectory was collected.
  • ρt = min(ρ̄, π(at|st) / μ(at|st)) — Truncated importance weight. Corrects for off-policy bias; clipped at ρ̄ (typically 1.0) to limit variance.
  • ct = min(c̄, π(at|st) / μ(at|st)) — Trace-cutting coefficient. Controls how far back temporal differences propagate. Clipped at c̄ (typically 1.0).
  • δtV = ρt(rt + γV(xt+1) − V(xt)) — Importance-weighted TD error.
  • Throughput — IMPALA's advantage: actors never wait for the learner. 4,096 parallel sims can feed one GPU learner.
Policy lag (steps behind): 10    ρ̄ clip: 1.0

SAC (Soft Actor-Critic)

SAC (Haarnoja et al., 2018) adds entropy regularization to the actor-critic framework: the agent is rewarded not just for high returns, but for maintaining high-entropy (diverse) action distributions. This creates a natural exploration mechanism and prevents premature convergence to deterministic policies.

J(π) = E[∑t γt (rt + α H(π(·|st)))]

Key terms:

  • H(π) = −E[log π(a|s)] — Entropy of the policy. High entropy = spread out probability = more exploration. Low entropy = peaked = exploitation.
  • α (alpha / temperature) — Controls the exploration–exploitation tradeoff. High α favors diverse actions; low α favors high-reward actions. Can be learned automatically.
  • Dual Q-networks — SAC trains two Q-functions (Q1, Q2) and takes the minimum. This prevents overestimation bias that plagues single-critic methods.
  • Soft Q-value — Qsoft(s,a) = r + γ E[Q(s',a') − α log π(a'|s')]. Includes the entropy bonus in the value estimate.
  • Reparameterization trick — Instead of sampling a ∼ π(s), sample noise ε ∼ N(0,1) and compute a = μ(s) + σ(s)·ε. Makes the gradient flow through the sampling step.
  • Off-policy — Like IMPALA, SAC uses a replay buffer. Old transitions can be reused because the Q-function is learned separately from the policy.
Temperature α: 0.20

Behavioral Cloning (BC)

Behavioral Cloning is the simplest form of imitation learning: treat the expert's demonstrations as a supervised learning dataset and train a policy to mimic the expert's actions given the same states. No reward function, no environment interaction during training — just input→output regression (or classification).

LBC = E(s,a)~D[−log πθ(a|s)]

Key terms:

  • Expert / Teacher — The policy being imitated. Could be a human, a hand-crafted AI (like GOAP/oracle), or a stronger model.
  • Demonstration dataset D — Pairs of (state, expert_action) collected by running the expert. BC trains on this like any supervised learning problem.
  • Distribution shift — The fatal flaw of BC. The student only sees states the expert visits. At test time, the student makes a small mistake, visits a new state, makes a bigger mistake, and the errors compound. The expert never demonstrated recovery from mistakes.
  • Compounding error — Over T timesteps, BC error grows as O(T²). A 1% per-step error becomes catastrophic over a long episode.
  • When BC works — Short horizons, simple action spaces, or when combined with DAgger. Also useful as a warm start before RL fine-tuning.

DAgger (Dataset Aggregation)

DAgger (Ross et al., 2011) fixes BC's distribution shift by iteratively collecting new data under the student's own policy, then asking the expert to label those states. Each round, the student visits states it actually encounters at test time, and learns what the expert would have done there. The training dataset grows (aggregates) across iterations.

Di+1 = Di ∪ {(s, π*(s)) : s ~ πi}

Key terms:

  • Dataset Aggregation — Each iteration: (1) run the current student policy, (2) record states visited, (3) ask the expert for labels, (4) add to dataset, (5) retrain. The dataset aggregates across all iterations.
  • πi — The student policy at iteration i. In early rounds, this is close to the BC baseline. It improves each round.
  • π* — The expert/teacher policy that provides labels. In our case, the GOAP oracle AI that computes the "correct" action for any state.
  • Distribution shift fix — BC trains on expert states. DAgger trains on student states. Since the student will visit these states at test time, the training distribution matches deployment.
  • Error bound — DAgger reduces compounding error from O(T²) to O(T). After enough iterations, the student's performance converges to the expert's.
  • Mixing / β-schedule — Classic DAgger interpolates: πmix = βπ* + (1−β)πi. Start with mostly expert (β≈1), gradually shift to student (β→0). Smooths the transition.
  • Win filtering — A practical variant: only aggregate winning episodes, so the student learns from its own successful trajectories rather than all trajectories. Avoids learning from failures.
DAgger iteration: 0    β (expert mix): 1.00
